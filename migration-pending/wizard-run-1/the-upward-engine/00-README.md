# The Upward Engine — a seed

> Written 2026-06-27 from session d5fc8ad1, grounded in the wizard run-1 field report + the live state of recollection, Dragnet, and the company. **A seed, not a spec** — provisional, meant to grow. Open-future: extend it, don't close it.

## The one line
The wizard, Dragnet, and recollection are **one compositional engine run *upward*** — a catch-all that pumps large, untrustworthy raw material up through layers of concurrent small models into the company's brain, rendering it into typed/addressed/embedded knowledge, with a reduce-reasoner and ultimately **Tim** seeing the pattern and steering the next pass. A **custom UI** is the surface that makes that *seeing* cheap, spatial, and steerable — and building it by running the chains on the company's own corpus is the dogfood.

1. **[[The Upward Engine — vision]]** (`01-VISION.md`) — the unifying frame: one engine, two directions, three+ chains, one UI as the institutionalised "look."

3. **[[Upward Engine — dogfooding plan]]** (`03-DOGFOODING-PLAN.md`) — the recursive bootstrap: point the chains at the company's own sprawl first; build the UI by using it; the seed→real-system path.

## Why it lives here
This is the wizard run's reason for existing made explicit: the run was the by-hand prototype; this is what it seeds. The wizard outputs (this folder) are the first real data the UI will render; the ideated process (the Patterned-Visibility methodology doc) is the drafted sequence; this set is where they become a system.
