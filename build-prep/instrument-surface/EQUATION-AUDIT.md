## The gap list (sequenced — unblocked & central first)

### 1. THE COINCIDENCE SPINE — circle ∧ square agree only on the axes  ✅ BUILT (`ce04fcf`)
Seed §1/§2/§3/§9: the concentric rings pass through the axial grid points; those **coincidence points** (on
BOTH circle and square) = "the REAL / ratified / addressable… the shared reference frame, the ratified spine,
the common memory." The corner/diagonal points never coincide = the forbidden translation-loss.
- **Observed (code+engine):** the engine ALWAYS emits `rings == grid/2` (verified: raw 4/2, semantic 16/8,
  by_node_type 4/2, by_nucleation 16/8). `ringRadii` = k/rings·R; the dyadic axial grid points = (2j−d)/d·R.
  With rings=grid/2 these **coincide exactly** on the four axes. So the geometry is RIGHT — but nothing MARKS
  the coincidence points, so the single most central claim of the seed (§3, the fundamental tension) is invisible.
- **Gap:** mark, in the Both view, the points where the rings cross the two axes (circle ∧ square agree) as the
  ratified spine; let the eye SEE that agreement happens only on the axes and nowhere else.
- Unblocked, surgical, central. → **first beat.**

### 4. EDGES AS CANONICAL MOVES — translate / rotate / scale  🟡
