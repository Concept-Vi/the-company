## THE PIPELINE (composition's typed stages — ~80% REUSE; the net-new is the TRANSLATOR)
1. **① INGEST (code):** parse the bundle's HTML/CSS/structure → a raw element tree. (reuse: a parser; deterministic.)
2. **② TAG (light models — the #71 4B swarm, extraction-not-judgment):** per element, emit a TRANSLATION-CANDIDATE (typed): `{ source_ref, candidate_archetype? (from the CATALOG: graph·selector·instrument·diagram·spatial-material·decision-card), candidate_component? (atom·molecule·organism), token_snaps (style → DNA tokens, ★ LAW-AWARE per DNA's snap-to-DNA: gold-budget · foreign-hue-quarantine · equal-lightness — never nearest-value-only), legibility {name,is} (derived, human) }`. Topic-free description per element (CANNOT-miss); never filters.
3. **③ COMPOSE (big model — the cloud judge):** the candidates → a VALID archetype instance / component definition, conforming to the catalog + the schemas, REUSING existing registry types (dedup — no parallel type), binding sockets, resolving the layout via the resolver. The mapping (candidate → catalog): a node-graph element → graph; a pick-list → selector; a live dial/gauge → instrument; a flow → diagram; a material/reveal → spatial-material; a decision → decision-card; a composed box → an organism. Reuse-first: match an existing type before minting.
4. **④ VALIDATE (code — the factory's gate, REUSE):** the registry.validate + the archetype schemas + the DECORATION-BAN (a device with no real data doesn't ship) + legibility-required (name/is or fail-loud) + the operator-laws. Fail loud — never a silently-wrong asset.
5. **⑤ RECONCILE (RHM + Tim, the operator-cycle — REUSE the decision-surface):** an ambiguity (which type? a new kind?) becomes a DECISION-CARD the RHM walks through with Tim → his answer writes back. The Face Pipeline's reconcile IS the decision-surface (one mechanism, two jobs).
6. **⑥ REGISTER (code — REUSE):** vi-vision:// address + corpus + resolution + persist → LIVE (the keystone renders it, DNA themes it, the company resolves it).
