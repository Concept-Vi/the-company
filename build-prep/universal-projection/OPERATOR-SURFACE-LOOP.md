## THE PHASES (sequenced so the loop can't build on sand ‚Äî but in sprint mode they overlap)

### Phase 0 ‚Äî THE OPERATOR JOURNEY  ¬∑  ‚úÖ APPROVED by Tim ("yeah OK", 2026-06-17)  ¬∑  Part 1
The design ground everything else conforms to.

**The spine (the discovery):** every moment, at its own depth, answers the SAME three questions ‚Äî
**‚ëÝ What am I looking at? ¬∑ ‚ë° What can I do here? ¬∑ ‚ë¢ Where can I go from here?**
Pedagogy = never leave a moment with any of the three unanswered, in human terms. **Drill-down = the same three
questions one level deeper** (composition's "deeper coordinate on the same axes") ‚Äî self-similar all the way down.

**The walk** (and the data each moment requires ‚Üí how the journey names the fields):
- **0 ¬∑ First contact (cold).** One breath: *what this whole thing is* + *what it's aimed at now* (meaning, not
  mechanism). ‚Üí the instrument's one-line identity + the current **address**'s *what-this-is / what-fills-it*.
- **1 ¬∑ Reading the whole.** What position means here ‚Äî centre, near/far, the slices ‚Äî for THIS aim + THIS lens.
  ‚Üí the **lens**'s *what-position-means-here* (centre/angle/radius/poles ‚Üí human).
- **2 ¬∑ Something catches the eye.** "What's *that*?" ‚Äî reward the wonder. ‚Üí each **element**'s identity on inspection.
- **3 ¬∑ The first reach (act).** Touch / hide labels / slide time; controls read AS controls; first act obvious, safe,
  reversible. ‚Üí each **control**'s *what-this-does* + the operator-state that remembers what they touched.
- **4 ¬∑ Going in (drill).** Re-aim at a thing's interior; re-orient with the same three questions at the new level.
  ‚Üí the SAME meta on the deeper **address** (every level self-describes).
- **5 ¬∑ Realising it re-aims.** It's not one view ‚Äî point elsewhere, dial other lenses. "You are here; here's what else
  you can aim at + what each lens shows." ‚Üí human names for every **destination** + every **lens** (what it reveals).
- **6 ¬∑ Coming back (time + state).** It remembers where they were + shows what changed. ‚Üí the operator's
  **journey/state** as itself addressable + self-describing.

**The legibility-type's facets (what the journey names ‚Äî composition builds the vessel, the journey shapes the set):**
- **address/thing** ‚Üí name ¬∑ is ¬∑ fills ¬∑ why
- **lens** ‚Üí what position means ¬∑ what it reveals
- **element** ‚Üí identity on inspection
- **control** ‚Üí what it does
- **destination** ‚Üí name ¬∑ what's there
- **journey/state** ‚Üí where you are ¬∑ where you've been ¬∑ what changed

Not one flat block ‚Äî these *facets*, each attached to the kind of thing it describes, all answering ‚ëÝ‚ë°‚ë¢ at whatever
depth the operator stands. Composition's "the field-set is itself a TYPE" ‚Üí the journey designs that legibility-type.
