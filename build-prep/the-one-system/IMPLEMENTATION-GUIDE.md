## Orientation (every session, before building)
1. Read AGENTS.md → MAP.md → STATE.md → THE-CONTAINER.md v3 → the organ study for your lane → COMPLETION-CRITERIA.md for your lane.
2. Check MERGE-COORDINATION.md + the board for parallel-session claims. ④ owns: schema `container`, new registries (types/, cascades/, edge_kinds/), the `project://` resolver branch, additive ledger columns, the mark-backing shadow. ④ does NOT touch ②'s code:// seam — propose via the board.
3. Announce lane starts on the board (cc_board item, scope the-fusion).

## The standing patterns (use these, never invent parallels)
- **Registry row**: file-per-row, id==filename, per-dir AGENTS.md, discovered not imported, malformed fails loud. Model: mark_types/, roles/, item_types/.
- **The gate**: create(kind=…) → render → validate in tempdir → write → git commit (reflect the dir's AGENTS.md same commit) → rediscover. Model: mcp_face/tools/create.py.
- **Shadow-then-flip**: new read path runs beside the old, divergence logged loud, authority flips only after proven-equal, old path COMMENTED OUT with the breadcrumb (never deleted). Model: runtime/grants.py + NORTH-STAR ①'s decisions.
- **Breadcrumb format**: "expected <new place>; previously <old place>; fix: <command>". Every resolve miss, every commented-out path.
- **Migration script shape**: ops/*.py, idempotent upserts keyed on source uuid, resumable, emits a reconciliation report (source→landed→excluded, sums match), never mutates cvi_mine. Model: ops/migrate_vectors_to_supabase.py.
- **Acceptance**: tests/<lane>_acceptance.py; run BEFORE commit; a red test = no commit (rule 10).
- **DB migrations**: numbered SQL beside 0011_ledger.sql, idempotent (IF NOT EXISTS everywhere), fresh-checkout reproducible.
