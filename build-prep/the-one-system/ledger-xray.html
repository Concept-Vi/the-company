Four findings

01

No ingest boundary

88% noise

The self-model swallowed everything — conversation transcripts, our own build scratch, channel data, vendored handoffs, binary assets. A self-model should index structure; data and scratch should be linked, not entries.

02

The extractor under-captures symbols

JS/JSX 100% flagged

Across all 1,304 audited code files, the deterministic extractor misses symbols in nearly every one — Python 96% flagged, JS/JSX 100%. The front-end languages are weakest. Fixable per-language; the audit re-confirms convergence.

03

The wiring is mostly broken links

66% dangling

30,620 of 46,613 edges point at an unresolved target — mostly calls the extractor saw but couldn't link to a definition. The graph knows a call happened, not to what. Needs a symbol-resolution pass.

Code is clean; the orientation layer isn't

code ✓

Embedding every file's description: the code barely duplicates (a handful of vendored copies). But the orientation layer was rebuilt every session — 20+ distinct *-MAP.md, multiple HANDOFF/STATE docs — and the "doc duplication" (811 files) is overwhelmingly channel-memory data ingested as markdown. Same boundary problem, seen twice.

The way out — what the ledger reveals

Highest leverage

Draw a typed ingest boundary

Index structure — code, capabilities, real vault docs (~1,700 entries). Link data, transcripts, scratch & assets instead of ingesting them. Cuts the ledger 11,292 → ~1,700 and fixes the system's own recall.

Then

Fix the per-language extractors

JS/JSX first (100% flagged) → re-extract → let the coverage audit re-confirm convergence.

Then

Resolve the dangling edges

A symbol-resolution pass that links calls/imports to their definitions — turn 66% broken references into real wiring.

Then

Consolidate the orientation layer

One live map, AGENTS-per-folder as the constitution; archive the 20+ session MAP/HANDOFF/STATE docs out of the live self-model.

Finally

Re-index the corpus on clean structure

So "ask the codebase" returns code, not planning scratch.

The throughline. The ledger is the seed of the one system — and it shows unification isn't "rewrite everything." It's draw the boundary, fix extraction & resolution, consolidate the map. ~1,300 distinct code files under ~10,000 entries of un-bounded ingest.
