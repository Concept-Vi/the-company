# Operator Surface — IMPLEMENTATION GUIDE (how to make each criterion true)

How to build each subsystem. Principles · sequence · do's/don'ts · file paths with roles · pseudocode. Tense is exact: IS = exists; SHOULD/WILL = planned. Evidence: see RESEARCH_SYNTHESIS.md; criteria: see COMPLETION_CRITERIA.md.

## Cross-cutting principles (apply to every section)
- **Ride, don't re-derive.** The bridge IS the one warm brain. The surface is a pure client of `/api` + `/api/stream`. Never open a 2nd FsStore, never re-implement a Suite method client-side. If a feature seems to need a backend, it's a bridge endpoint — find it (RESEARCH_SYNTHESIS Round 2) or it's a flagged net-new bridge addition.
- **Fuse the quarries, leave the rest.** Harvest parts per S15. Foundation = surface/app's thin-client spine + fine-paper tokens + layout switch. Substrate = canvas/app's address spine + SSE wiring + decision/consent components. NEVER carry the tldraw-graph-as-surface or the DNA live-load.
- **Fix the look centrally.** The look of both frontends lives in `design/_system/tokens.json` (+ the claude-ds axes merged in). Do NOT style surfaces with bespoke values — build from the design system's tokens/components. This is the FORM bar; the design-lint enforces it.
- **Translate to human meaning.** Every surface renders through the RHM (`/api/up-translate`); Tim never sees machine names/addresses. A navigable visual/spatial surface, not a text-wall.
- **No-silent-failure.** Every failure → a human-register Notice + a (Gap) note. The `getJSON`/`jr` fail-loud client (both apps have one) is the model — keep it.
- **Convergence/widen scope.** When a surface needs something the centre lacks, fix it in the centre (the 5 company-improvements are the first instances). Don't work around.

## Form (the design rubric — applied to EVERY surface, half of done)
Built on the design system's tokens/components (NO hardcoded values, NO bespoke one-offs) · no overlaps · responsive at desktop/portrait/landscape (the `classify(w,h)` switch) · consistent scale/type/spacing · settings consolidated · a navigable visual/spatial surface, not a text-wall/list (Tim recognises by sight) · empty/loading/error states present and in human register · the outcome demonstrable by use. A surface is FORM-green only when a design-critic agent passes it against this rubric and the design-lint passes.
