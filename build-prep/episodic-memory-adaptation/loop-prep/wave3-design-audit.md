# Wave-3 ADVERSARIAL DESIGN AUDIT — recollection loop-prep

> **What this is.** A skeptic's pass over the drafted loop-prep triad (`COMPLETION_CRITERIA.md`, `IMPLEMENTATION_GUIDE.md`, `RESEARCH_SYNTHESIS.md`) + `OPEN-DECISIONS.md`, checked against the governing laws (`brain/CONVERGENCE-OBJECT.md`), the 38-constraint ledger (`wave1-constraints-ledger.md`), the decided design (`EPISODIC-UPGRADE-PLAN.md`), and the code-grounded merge frame (`MERGE-INTENTION.md`). The job is to find what's wrong, missing, or law-violating — **not** to confirm the docs are good. Where the docs are right, I stay silent; every entry below is a defect, a gap, a risk, or a decision Tim hasn't been given.
>
> **Evidence tags:** **[Observed]** = read directly in a doc/the code-grounded reads. **[Inferred]** = my reasoning about a consequence (correctable). Each finding is keyed to the doc + section and is phrased as an action.
>
> **Severity:** 🔴 serious (a pillar promise is unbuildable from the specced parts, OR a law is violated/silently dropped) · 🟠 real gap (a decided design part has no criterion, or a criterion can be passed by code-existence) · 🟡 hygiene (a missing Form face, an under-specified seam, a tightening).
>

## A. THE MOST SERIOUS FINDINGS (read these first)

### 🔴 A1 — The net-new "merge/omniscience" build list from MERGE-INTENTION is NOT carried into the triad as criteria. Pillar 2 is asserted, not specced.
**[Observed]** `MERGE-INTENTION.md` §SYNTHESIS names a precise net-new list — the things that *do not exist yet* and must be built for the pillars to be real:
1. the cross-system **join code** (the spine is queryable but *nothing wires it*);
2. **`tool_result` parsing** (the entire output axis is 100% NULL — "where half the causal story lives");
3. **true dedup/merge/prune consolidation** (substrate's `consolidate` is *aspirational* — wikilink-resolve + recount only);
4. **`find_relations` edges** (the graph faculty is registered-but-EMPTY: relation *types* exist, zero edges, no edge table);
5. the **re-embed migration** to one space (the 384-dim vectors are not reusable).

The triad handles some of these well (re-embed = G3.4; edges = G4.2/links/ schema; sidechains = G1.1). But two are **under-acknowledged in a way that lets the build think the work is "harvest" when it is "build from zero":**

- **`find_relations`** — IMPLEMENTATION_GUIDE §5 says "Build the `find_relations` edge layer (unbuilt in the Company — recollection builds its own, portable)" which is correct, BUT COMPLETION_CRITERIA G4.2 ("Semantic links") is the only criterion that exercises cross-project concept links, and it is phrased as a *retrieval* check ("a cross-project concept link connects the same concept in ≥2 projects") with no criterion that the **edge-construction at corpus scale** actually ran and produced a populated edge graph. The hardest, load-bearing Pillar-2 build (populate a graph that has never had a single edge) has no FUNCTION criterion of its own. **Action:** add a criterion: *"the concept-link inferer runs over the real corpus and produces a populated edge table (count > 0, spot-checked for correctness); `what_projects_include(X)` traverses real edges, not a query-time recompute."* Without this, G4.2 can be "passed" by a single hand-made edge.

- **True consolidation/dedup/merge** — IMPLEMENTATION_GUIDE §9 correctly flags "true dedup/merge/prune is NET-NEW (memoirs reference)" and G9.2 (annealing) covers "folds duplicates." But the **merge-as-join across two projects** — the literal "merge these two projects" Pillar-2 promise — is only in G5.3 (questionless-sweep, "merge-as-join MATCHED/build/debt list"). That is a *findings list*, not a merge. **[Inferred]** Nothing in the triad delivers an actual project-merge *operation*; the design (EPISODIC-UPGRADE-PLAN, P1/§Foundation) only ever promises that recollection lets *an agent* do the merge by giving it the MATCHED/divergence inventory. **This is fine IF stated** — but the COMPLETION_CRITERIA's own G10.3 says the omniscience test includes *"merge these two projects" returns correct cross-project … answers*, conflating "produce the inventory an agent merges from" with "merge." **Action:** split G10.3's "merge" clause: the deliverable is the **decision-grade merge inventory** (MATCHED concepts / build-list / debt-list / divergences across the two projects' real histories), explicitly NOT an automated code-merge. Otherwise G10.3 is unpassable-as-worded and invites a green-paint.

### 🔴 A2 — Pillar 1's "agent already understands what Tim means" rests entirely on the principle/concept corpus existing — but the criteria let the switch-flip test (G10.2) pass on a near-empty corpus.
**[Observed]** G10.2 (switch-flip): "an agent loaded with recollection's floor answers a 'what does Tim mean by X / position on Y' correctly … where a cold agent fails." **[Inferred]** This is the single most important criterion in the whole build (it IS Pillar 1). As written it can be satisfied with **one** ratified principle and **one** lucky X. The North Star (ledger) is "an agent that needs no re-explanation" — a *population* property, not a single hit. The triad has no criterion establishing **floor coverage**: that the floor injected at session start actually spans Tim's *active* principle set (the things he repeats), not a token sample. **Action:** strengthen G10.2 (or add G10.2b): the switch-flip test is run over a **Tim-supplied or correction-log-derived panel of ≥N "things I shouldn't have to re-explain"** (candidates: the named frustrations and corrections already in MEMORY.md — no-versioning, bge-zero-priority, extraction-vs-judgment, not-a-developer, verify-by-use, …), and the floored agent must clear the panel where a cold agent fails. Tie the panel to `tim_correction` records (convergence der.7) so it's grounded, not invented. Mark this the keystone proof.

### 🔴 A3 — "Pool a concept across ALL its dispersed times" (the defining Pillar-2 retrieval mode) has a FUNCTION criterion that under-tests the hard part: completeness/recall, not just "returns a pooled arc."
**[Observed]** G5.2 ("Two modes"): "gather-all-and-aggregate returns a pooled arc for a much-discussed concept." G6.1/G6.2 cover folding dups and currency. **[Inferred]** The pillar bar (ledger Pillar 2) is *"retrieval must POOL a concept across all its dispersed occurrences, never single-hit."* The risk: gather-all-and-aggregate can return *a* pooled arc that silently misses half the occurrences (a single lens's top-k union), and the criterion would still pass — a **silent recall failure**, which is exactly the B6 (no-silent-failures) trap applied to retrieval quality. There is no criterion that the pool is *complete* (or that incompleteness is *surfaced*). **Action:** add to G5.2/G5.3: a pooled-concept query must either (a) demonstrate recall against a known-complete ground set (pick a concept with a countable set of source exchanges; show the pool finds them), or (b) **emit a coverage/confidence verdict** ("pooled N occurrences across M projects/sessions; K low-confidence") so missing-ness is visible, not silent. This is the map(all)→reduce path (sweep), and it's why the sweep lane (G5.3) — not query-time top-k — is the real Pillar-2 organ; the criteria should say so explicitly and make the sweep, not gather, the omniscience proof.

### 🔴 A4 — Grounded-chain law (B2) is asserted in FORM language but NOT made verifiable: no criterion proves "the grounded path is the AI's *easiest* path" / "the cold agent *cannot* rebuild the ungrounded version."
**[Observed]** B2 (ledger) and IMPLEMENTATION_GUIDE §0 both state the grounded chain "isn't done until it's the AI's EASIEST path — saved/runnable through the MCP as one thing, so the next (cold) agent … *cannot* rebuild the ungrounded version." COMPLETION_CRITERIA G7.1 FORM says "the grounded path is the *easy* path (grounded-chain law)" and G7.2 (skill) wraps the flow. **[Inferred]** But "easy path" is asserted, never *tested*. The law's teeth are the negative: a cold agent handed the MCP should *fall into* the grounded production line and *not* be able to easily run the raw ungrounded query as the default. No criterion checks that the raw/ungrounded path is a *deliberate extra hop* (it's claimed in P7-D2 but not verified), and the `search` name is KEPT (G7.1) — which is the *blunt ungrounded* tool. **Risk:** keeping `search`/`read` as drop-in names (good for compatibility) silently re-exposes the ungrounded flood path as a first-class verb, violating the very law the rest of the surface upholds. **Action:** add a criterion: a fresh agent, given only the MCP tool list + descriptions, drives a real "what's Tim's position on X" task and **chooses a grounded axis-tool unaided** (handles+verdicts), and the raw-flood path requires an explicit override/extra hop — verified by an actual cold-agent run (this is also B5's "fresh agent drives a real task unaided" bar, which currently has no criterion at all — see B5 below).

### 🔴 A5 — The judgment layer's central judge (D-2) is unresolved, yet multiple criteria (G2.4 ratification, G6.2 jury, G10.x proofs) depend on which seat fills it — and the unattended path may be silently unbuildable standalone.
**[Observed]** D-2 (OPEN-DECISIONS) is open: central judge = session-model vs local-big-model vs both-by-mode (my lean). G2.4 (principle ratification) explicitly says "with the powerful-judge slot (D-2)"; the dream-phase (G9.2), background sweeps (G5.3), and unattended supersession (G6.2) all need an **always-available** judge. **[Inferred]** If D-2 resolves to "session model only" (option a), then **every unattended/background judgment criterion (G9.2 annealing, G5.3 scheduled sweeps, background L3 ratification) cannot run when no session is active** — they'd be silently blocked. The triad doesn't flag this dependency: G9.2 and G5.3 are written as if they just run. Per the rules-of-engagement (an item blocked on an unanswered decision stays ☐ and leaves the seam), G9.2/G5.3/G2.4 should be **marked blocked-on-D-2**, and the implication ("no local big judge ⇒ no unattended judgment") should be on Tim's sheet as a consequence, not buried. **Action:** annotate G2.4, G5.3, G6.2, G9.2 with "blocked-on-D-2 (unattended judgments require option b/c)"; surface the consequence in OPEN-DECISIONS D-2 (it currently lists options without stating that option-a kills the unattended lane).

### 🔴 A6 — `tool_result` is 100% NULL and the parse pass is net-new — but the design's "lazy fetch on demand" assumes the result is *fetchable*, and the criterion (G4.3) never verifies the source actually yields it.
**[Observed]** RESEARCH_SYNTHESIS 1.1 + MERGE-INTENTION §CAPTURE: `tool_result` is 100% NULL in the live DB; the values survive only in the raw `.jsonl` archive and parsing them is **net-new**. P6-D1 / IMPLEMENTATION_GUIDE §5 say tool_results are LAZY: "keyed to the crossing, fetched on demand from the raw source." G4.3 FUNCTION: "a tool_result is NOT in the distilled memory but is fetchable on demand via its key." **[Inferred]** "Fetchable on demand" silently assumes the raw `.jsonl` is (a) still present, (b) addressable by the crossing key, and (c) parseable to the specific result. None of that is built or verified by G4.3 as worded — it can pass by returning a *key* that *points* at a result that was never actually retrieved. This is a B6 silent-failure surface (the agent asks "what came back from that action," gets a dead key, and the system looks like it answered). **Action:** G4.3 FUNCTION must verify the round-trip: a crossing key resolves to the *actual result content* pulled live from the raw archive on real data — not just that a key exists. Add the raw-archive-availability precondition (the archive is mirrored Company-side, not only in the version-clobbered plugin cache — already G0.1, link them).

## B. LAW / CONSTRAINT VIOLATIONS & SILENT GAPS (audit question a)

### 🟠 B-law-1 — Classification Law is named in G5.1 FORM but never *enforced* as a verifiable property; no criterion catches a >5-bin or non-even-spaced type.
**[Observed]** CONVERGENCE-OBJECT §II + Classification Law: any classifier type must declare 1 axis / 2 extremes / 3–5 even bins / progressive chains over capacity. G5.1 FORM: "classification obeys the Classification Law (1 axis/2 extremes/3–5/chains); types are registry rows." **[Inferred]** This is stated as a property of the *output* but there's no **deterministic floor** that *rejects* a registry-declared type that violates it (e.g. a 9-way single classifier). Per B2 (grounded-chain), a law without a deterministic floor that enforces it is "not designed." **Action:** add a FORM check (or G0.2 schema check): the `unit_type`/classifier registry **validates on write** that each classification type declares ≤ capacity bins with even spacing, fail-loud on violation (B6). Prove by attempting to register an illegal 9-bin type and watching it be rejected. This also operationalizes derivation 4 (types born at deep holes) — illegal-bin pressure is the signal.

### 🟠 B-law-2 — "Edges must be directional with a meaningful inverse" (Classification-Law sibling) is in the links schema but has no verification that an undirected/inverse-less edge is *rejected*.
**[Observed]** IMPLEMENTATION_GUIDE §1 links schema: "Edge types are directional-with-inverse (precedes/succeeds, contains/is-contained-by, …)." CONVERGENCE-OBJECT §II: "free-floating labels and undirected relations are **illegal** because they have no coordinates." **[Inferred]** No criterion checks the *illegality* is enforced. **Action:** G4.x FORM: registering an edge_type without a declared inverse/direction fails loud; prove by attempting it.

### 🟠 B-law-3 — Convergence Law / "no valid unit at a deep hole" has a health *scan* (G9.1) but no criterion that the system *acts on* a deep-hole unit or that the resistance metric is the real structural∩semantic distance.
**[Observed]** CONVERGENCE-OBJECT §I + der.2: resistance = distance(structural-position, semantic-position); "a well-formed system places no valid unit at a deep hole"; der.4: persistent non-convergence = a missing type. G9.1 FUNCTION: "a real per-unit resistance score + a heat-map of misfiled/drifted units." **[Inferred]** G9.1 verifies a score is produced; it does NOT verify the score is computed *the specified way* (embed the address/type-description with the steered lens vs embed the unit) — it could be any distance and pass. And der.4 (new types born at deep holes) — the system's stated **growth law** — has **no criterion at all** (it's the most-[GEN], awaiting Tim's fit-test, but it's in the design as derivation 4 and EPISODIC P5/§health references it). Per A3/A4 (no-MVP, no-deferral) it can't just be absent. **Action:** (1) G9.1 FUNCTION must specify and verify the *two-embedding* resistance definition. (2) Add a criterion (or an explicit OPEN-DECISION) for derivation-4 growth: either build "persistent deep-hole units surface a candidate new type" or record it as a Tim-gated open item — do not let it silently vanish from the criteria when it's in the doctrine.

### 🟠 B-law-4 — No-silent-failures (B6) is honored in prose but only ONE criterion (G3.3 "no silent fail" on nomic-code pooling) actually tests a fail-loud path. The other named silent-failure traps have no negative test.
**[Observed]** B6 (ledger): every op succeeds or fails LOUD; named traps include "lens unavailable, parse fail, model not loaded → surface + record a gap, never silently return empty or quietly fall back to a weaker lens." Only G3.3 tests a silent-fail trap. **[Inferred]** The highest-value silent failures for *this* system are **retrieval-completeness** ones (A3) and **fallback-to-weaker-lens** ones (D9 make-each-thing-work). None are tested. **Action:** add a cross-cutting verification (a FORM rider on G3.x/G5.x): kill a lens / unload a model mid-run and verify the system **surfaces a Notice + records a `(Gap)`**, and does NOT silently return the degraded result as if complete. This is a concrete, runnable adversarial test and it's currently missing.

### 🟠 B-law-5 — No-fiction-about-Tim: G10.4 ("zero fabricated principle records on a sample") is good but the *abstention* behavior (the law's actual mechanism) is untested.
**[Observed]** No-fiction law (CONVERGENCE der./ledger): "uncertain → abstain/flag, never fabricate; empty is `""`, uncertain abstains." G2.2 FORM "no-fiction verified on a sample"; G10.4 "zero fabricated records on a sample." **[Inferred]** Both test the *positive* corpus (what got written is real). Neither tests the *negative*: that on a deliberately ambiguous/insufficient source the extractor **abstains** (returns empty/flag) rather than inventing. A no-fiction system that never abstains is just a system that got lucky on the sample. **Action:** add to G2.4/G10.4: feed an adversarial source that *looks* like it contains a Tim-principle but doesn't (or is genuinely ambiguous); verify the extractor abstains/flags rather than fabricating — the abstention path is exercised and observed.

### 🟡 B-law-6 — Extraction-vs-judgment (B3) is structurally honored, but no criterion *proves a worker cannot emit a verdict*.
**[Observed]** B3: workers extract, central judges; "workers must be explicitly forbidden from emitting verdicts." IMPLEMENTATION_GUIDE §3 Do/Don't says it; no criterion verifies the membrane. **[Inferred]** This is the exact error Tim already corrected — it deserves a guard, not just a prose rule. **Action:** G2.x or G6.x FORM: the worker (4B extractor) output schema is verdict-free by construction (structured-observation-only); attempting to route a worker output as a verdict fails / is caught. Light, but it closes the one error the law exists to prevent.

### 🟡 B-law-7 — Not-agent-architecture-by-default (B4): the sub-agent (G7.3) is the governed exception, but the criteria don't assert the *default* path is dataflow (resolve→work→persist→trigger), risking the build drifting agentic.
**[Observed]** B4: LLM use defaults to content/variable-resolution; agentic is the governed non-default. The triad's gather/distill/judge are described as pipelines (good), but G7.3 (sub-agent) and the "active loop" framing could invite an agent-loop default. **Action:** a one-line FORM rider on G5/G6: the everyday gather→judge path is a *dataflow composition* (no autonomous decide-act loop); the looping sub-agent (G7.3) is invoked only for the deep arm. Cheap insurance against the framework-drift Tim rejects.

### 🟡 B-law-8 — No-fiction / grounded-chain require a **context package** per model step (B2: exemplars + closed vocabularies + ground summaries + parent records). The triad never lists the context-package as a deliverable for the distill/classify/judge steps.
**[Observed]** B2 (ledger): "A model-judgment step is only *designed* when its CONTEXT PACKAGE is designed with it." The criteria check outputs (records, verdicts) but not that each step *ships its context package*. **[Inferred]** Without this, the build can produce a worker prompt with no closed-vocabulary/exemplars and still pass the output criterion — then drift. **Action:** add a FORM rider on G2.2/G5.1/G6.2: each model step's registry row includes its context-package (exemplars + closed vocab + ground summary refs); verify one step's package is present and used.

## C. PILLAR-PROMISE vs DELIVERED-DESIGN GAPS (audit question b)

### 🔴 C1 — "Agent already understands what I mean" depends on the FLOOR injection (G8.1) being a *good codebook*, but G8.1 only verifies *that something is injected*, not that what's injected is the right, current, compact identity.
**[Observed]** G8.1 FUNCTION: "a fresh session's agent already holds Tim's active principles + this project's standing state, injected by a SessionStart hook, WITHOUT being asked." **[Inferred]** This passes if *any* principles are injected. The pillar needs: (a) the floor is **current** (superseded principles excluded — depends on G6.2 supersession actually running at floor-build time), (b) **compact** (it's a codebook, not a dump — and there's a real recursion risk: the original episodic-memory's SessionStart hook was a recursion source the design explicitly kills by moving summarization local — IMPLEMENTATION_GUIDE §3; but a SessionStart *injection* hook that itself triggers gather→judge→(model) re-introduces a SessionStart-time model-call. **Is the floor injection precomputed/cached, or live at session start?** Not specified.) **Action:** (1) G8.1 must verify the injected floor is current (a superseded principle is *absent*) and compact (token-bounded), not merely present. (2) Resolve/flag whether floor injection is a **precomputed cached artifact** (refreshed by the deep-pass) vs **live gather→judge at SessionStart** — the latter risks both latency and a new recursion/again-at-every-start cost. This is a NEW open decision (see D-NEW-1).

### 🟠 C2 — "Knows each project's structure down to code level / all scales" (Pillar 2, ledger + EPISODIC §0) has weak criterion coverage at the *code/structure* scale.
**[Observed]** Ledger Pillar 2: "Knows each project's structure down to code level — all scales." Criteria: G3.3 (code lens "find code that does X"), G4.1 (artefact↔conversation crossing). **[Inferred]** There's no criterion for *"what is the structure of project Z"* or *"what was built in project Z and when"* answered at the **project-structure scale** from real history — G10.3 asserts it but no component criterion builds the scale-rollup that answers it. G2.3 (L3 rollups) covers "a project rolls up into an arc" but "arc" ≠ "structure down to code level." **Action:** add a criterion: a project-scale query returns the project's **build structure** (what components/artefacts exist, when each was first written, via the crossings ledger) — the concept↔structure↔code scale axis is *demonstrated*, not just claimed. This is the load-bearing "all scales" promise and currently only G10.3 (a north-star proof) touches it, with nothing building toward it.

### 🟠 C3 — "Merge these two projects" — the design delivers an *inventory for an agent to merge from*, not a merge. Honest, but the docs blur it (see A1). The deeper gap: the inventory's quality depends on cross-project concept-sameness links (G4.2) which depend on edges that have never existed (A1).
**[Inferred]** Chain: G10.3 merge ← G5.3 sweep merge-as-join ← G4.2 cross-project concept links ← the concept-link inferer populating an empty edge graph (net-new, A1). If A1's edge-construction criterion is missing, the whole merge promise rests on an unbuilt foundation that no criterion forces into existence. **Action:** make the dependency explicit in the priority order and add the edge-construction criterion (A1). The merge promise is buildable *only if* the edge layer is actually populated and verified at scale.

### 🟠 C4 — The "single unified capture path" (P10-D3 / G1.3 FORM "one unified path, not a 4th parallel lane") risks becoming exactly the 4th lane it forbids — and no criterion verifies the *old* lanes are retired/unified.
**[Observed]** P10-D3: collapse the 3 overlapping ingestion paths (episodic SessionStart hook · Supabase sync · lobe-importer) into ONE. G1.3 FORM: "one unified path, not a 4th parallel lane." **[Inferred]** recollection is *itself a new plugin with its own capture* — i.e., absent active retirement, it *is* a 4th lane sitting alongside the 3 existing ones. The criterion checks recollection's path is internally unified; nothing checks the 3 legacy lanes are actually decommissioned or subsumed. The "make each thing work, never route around" (D9) + "one continuous process" intent is violated if recollection just adds itself. **Action:** add a criterion (or explicitly scope-out with Tim): the legacy lanes are retired/subsumed, OR record that recollection-standalone *intentionally* runs parallel until absorption (D-8) and the unification is an absorption-time deliverable — don't let "unify the lanes" read as done when a 4th lane was added.

### 🟡 C5 — Interactive/co-mode distillation (G2.5) and proactive injection (G8) both depend on the Company *interface* and *modes registry* that EPISODIC P5-D2 itself flags as "not fully built."
**[Observed]** EPISODIC P5-D2: interactive distillation happens "through the Company's interface (not fully built)." G2.5 FORM "mode-gated via the modes registry"; G8.2 FORM "mode-gated." **[Inferred]** Standalone recollection has no Company interface/modes registry. These criteria are silently cross-dependent on an external unbuilt surface. **Action:** mark G2.5 and the mode-gating clauses of G8.2 as dependent-on-Company-interface; specify the standalone fallback (a local mode flag) or flag as absorption-time. Per no-silent-failures, the dependency shouldn't be invisible.

### 🟡 C6 — Voice/STT capture (P10-D1 "open to non-CC") and document/PDF modality (P9) are DECIDED in the design but have **no criteria at all**. No-MVP (A3) + no-deferral (A4) say they're in scope.
**[Observed]** P9 (modality: images BOTH, documents, voice→text) and P10-D1 (open to non-CC sources, voice→STT) are decided. COMPLETION_CRITERIA G1 (capture) is Claude-Code-only; no image/document/voice criterion exists; G3 lenses include a `visual` lens row but no criterion exercises an image unit end-to-end. **[Inferred]** This is the clearest no-MVP gap: decided design parts with zero criteria. The pragmatic read is "Claude Code is the immediate target" (P10-D1) — legitimate — but then it must be a *recorded sequencing*, not a silent omission. **Action:** either add criteria for the visual-unit round-trip (P9 images BOTH: VL fingerprint AND OCR/layout — at least one end-to-end) and a pluggable-source proof for one non-CC source, OR record on Tim's sheet that non-CC modalities are explicitly sequenced post-Claude-Code (A4: sequencing ≠ deferral, but it must be *stated*). Right now the visual lens (G3) is half-present (a registry row, no use), which is the worst state (looks covered, isn't).

### 🟡 C7 — "Gets better at predicting what Tim wanted" (Pillar 2, 5th requirement; EPISODIC §0; convergence der.7 — corrections as training signal) has no criterion and no learning-loop deliverable.
**[Observed]** EPISODIC §0 Pillar 2: "Gets better at predicting what Tim wanted to do." der.7: injection policy trains on `tim_correction`. G8.2 FORM "learns from `tim_correction`." **[Inferred]** "Learns from tim_correction" appears once as a FORM clause with no FUNCTION criterion that a correction *actually changes* future injection/retrieval. The predictive/model-of-Tim edge — a stated pillar requirement — is effectively unbuilt in the criteria. **Action:** add a criterion or record as explicit Tim-gated sequencing: a recorded `tim_correction` demonstrably alters a subsequent injection decision (the introspective loop closes once, observed) — OR flag that the learning loop is a named post-first-build component. Don't let a pillar requirement live only as a FORM adjective.

## D. COMPLETION-CRITERIA QUALITY: verify-by-USE & two-faced (Function + Form) (audit question c)

### 🟠 D1 — Several items can be passed by code-existence / single-instance, defeating verify-by-use. Catalog:
**[Observed/Inferred]**
- **G4.2 (semantic links)** — passable with one hand-made edge (see A1). Needs scale/population.
- **G5.2 (pooled arc)** — passable with an incomplete pool (see A3). Needs coverage proof.
- **G8.1 (floor)** — passable with any injection (see A2/C1). Needs current + compact + panel.
- **G10.2 (switch-flip)** — passable on one X (see A2). Needs the panel.
- **G3.1 (multi-lens)** — FUNCTION "a unit has multiple `(unit,space)` rows" is a *row-existence* check; the FORM (swap a lens via registry) is the real test. Move the swap proof into FUNCTION or it's code-existence.
- **G6.2 (jury routed)** — "on a clean recall it does NOT fire" is good (negative test, keep). But "rules current-vs-superseded" needs a real superseded pair from Tim's actual history, not a synthetic one — specify real-data.
**Action:** rewrite each flagged FUNCTION to require real-data + population/coverage, not existence. The pattern to apply everywhere: *"on real corpus data, demonstrating the hard property (scale/recall/currency), observed."*

### 🟡 D2 — Items missing or weak on their FORM face:
