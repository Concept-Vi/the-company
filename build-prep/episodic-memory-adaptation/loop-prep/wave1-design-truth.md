## PART A — THE CONSOLIDATED DESIGN (one coherent system)

### A0 · What recollection IS (the north star, not negotiable) [Observed — PLAN §0]
recollection is an **identity-continuity layer**, not a transcript search box. Its primary job: make an agent **already understand what Tim means and probably wants**, carrying his described principles/concepts forward across all projects and sessions so he never re-explains himself. Specific-instruction recall is the *exception*, not the default. It feeds the same target as `model_of_tim` / `foundation/TIM.md` / the principles.

**Two pillars, ONE mechanism (the single-origin transcript spine):** [Observed — PLAN §0]
1. **Identity continuity** — agents decode against Tim, never the developer-prior; the persistent codebook.
2. **Cross-project omniscience** — the reconstructable total history of everything Tim ever built (WHAT, WHEN, structure, down to code-level), able to pool a concept across all its dispersed occurrences in time, and do broad cross-project ops ("merge these two projects").

Both pillars are the SAME rollup flow read at different heights: distill (meaning/concepts) + provenance (what/when built) + scale axis (concept↔structure↔code) + temporal axis + pool-across-time → reason → act. **The conversation store is the single primary source; all code is a lossy, never-reviewed AI projection of it.** [Observed — PLAN §0, MERGE-INTENTION lens]

### A1 · Base [Observed — PLAN §0]
Clone-as-sibling plugin **`recollection`** — own data dir (survives plugin updates), own skills, own custom sub-agents (distinct from episodic-memory's `remembering-conversations`/`search-conversations`). Standalone now, **absorbable into the Company later** (NOT wired into Company pgvector yet — that waits on the merge). Same MCP tool surface so it's usable by me + other Claude Code agents.

### A2 · The data model (units + links + fingerprints over atoms) [Observed — PLAN P1, P6, THE COMPLETE DESIGN]
- **Atoms** = events (message / tool_call / result) on a timeline — raw bedrock, nothing lost.
- **Units** = a **typed REGISTRY** of multi-scale units (NOT one atom): principle/concept statement · turn-context work unit · conversation/session/project/intent-constellation rollups · … Chosen by the query's axis. Each unit carries **addresses in two directions**:
  - **UP (containment):** `unit ∈ session ∈ project` — and this is **free/certain from the transcript**: every session physically runs in a working directory = the project folder (the cwd, confirmed present on the exchange). The scale hierarchy is solid ground, not inferred. [Observed — PLAN P1]
  - **ACROSS (aboutness):** to the topic / concept / artefact it relates to.
- **Links (provenance graph)** = **mechanical skeleton** (containment + crossings — exact/free, asserted as fact) + **semantic enrichment** (cause-edges, cross-project concept-sameness — model-made, so TYPED + confidence-graded; high-stakes links run the P4 judgment layer). **Identity = the connected graph; one continuous linking process.** [Observed — PLAN P6]
- **Fingerprints** = multi-coordinate; each unit has many semantic positions (the lens set below). Steering vocabulary **MINED from Tim's corpus**, not dev defaults. Models = swappable registry slots.

This makes recollection answer **relational/address queries** ("what projects include X", "when has X been a problem", "where has X been described", "when was this part worked on in this project"), not just content search. [Observed — PLAN P1]

### A4 · The pipeline (the flow) [Observed — PLAN THE COMPLETE DESIGN]
1. **CAPTURE (P10)** — TOTAL: all CC sessions **incl. agent sidechains** (NON-NEGOTIABLE — where most of Tim's work happens; existing lanes skip them), all projects, open to non-CC sources; backfill (bootstrap from the ~13,270-conv archive) + live streaming; ONE unified pluggable capture path (collapse the 3 overlapping lanes — episodic SessionStart hook, Tim's SessionEnd→Supabase sync, lobe-importer).
2. **DISTILL (P5)** — 3 layers, all grown/non-hardcoded:
   - **L1 Summary** (per-conversation "what was this about"; finish all 13,270 via the 32-wide swarm — current system stalled at 1,267).
   - **L2 Structured extraction** (TYPED records: decision · rationale · tim-correction · my-error · bug-fix · needs-tim · frustration · pattern · principle · concept-definition · what-was-built — each linked to source + embedded). This is the search-box→understanding jump = Pillar 1, mechanically.
   - **L3 Rollups** (compose L2 UP across scale → project-arc → "Tim's settled view on concept X, pooled from everywhere"). **What survives the rollup at every scale IS a principle** (= where model_of_tim + steering vocabulary come from).
   - **Triggers:** backfill + live + deep-pass + **INTERACTIVE** (in directed/co modes, through the Company interface) — trigger set itself extensible.
   - **Principle gate:** candidate-staging → **powerful-model ratification WITH Tim** (foundation-exchanges pattern mechanized; powerful-model seat = me-via-MCP OR in-app). **Gate is ONLY for principle/identity-level declarations**; non-principle L2 records self-assemble without the gate. [Observed — PLAN P5 D3 confirmed]
3. **PLACE/EMBED (P2)** — the lens set above.
4. **LINK (P6)** — build the provenance graph; **tool_results kept LAZY** (keyed, fetched on demand, NOT distilled — don't spend the swarm on 6.7M action-outputs); **forward anchoring = LINK don't embed** (a verbatim quote inside an artefact is a redundant copy that can drift — the live link IS the anchor). [Observed — PLAN P6 D2/D3]
5. **GATHER (P3)** — the querier is the AGENT, not Tim. Pass 0 structural/free → Pass 1 message-type classify (tiny model) → Pass 2 sentence-grain decompose → Pass 3 per-block typed classify at ~120 concurrency (Classification Law) → Pass 4 typed lookups by block-type → Pass 5 aggregate per thread → inject. **Two modes, both required:** top-k AND **gather-all-and-aggregate** (pooling across time = Pillar 2's mode). Fusion by RRF; **agreement-across-lenses = the relevance signal.** [Observed — PLAN P3]
6. **JUDGE (P4) — the judgment layer (general, not a rerank stage):** three cleaners — **Proofreader** (tiny CPU, always, scores question+1-candidate) → **Set-reader** (reads whole shortlist, folds dups, flags disagreements) → **Jury** (4B, 32-wide, reads candidates in time-order, rule-driven, wakes on high-stakes/conflict). **Judgments are an OPEN TYPED REGISTRY** (seeds: junk-kill · supersession-on-time-axis · set-curation arc · visual; +more declared not coded). **The jury is a GENERAL machine** — pointable at ANY judgment, not just relevance ("what should the agent ask?", directions, …). **The memory is ACTIVE** — runs its own follow-up retrievals (recall→read→notice gap→recall again→assemble). Verdicts are **self-explaining, typed, multiple-per-result** (relevance/currency/confidence each a typed record). [Observed — PLAN P4 D1/D2/D3]
7. **RECALL — three arms:** shallow/precise = **P7 axis-addressed MCP tools** (parametric core + named question-shaped front doors, registry-driven, handles-first + return-format override) · ambient = **P8 proactive injection** (CC hooks: SessionStart floor = THE KEYSTONE; UserPromptSubmit in-session; same gather→judge machine, mode-gated, learns from `tim_correction`; **pre-compaction snapshot DROPPED** — the memory IS the continuity) · deep = the **P4/P8 sub-agent** (the active loop). **P4's looping judge and P8's "memory sub-agent" are ONE organ.** [Observed — PLAN P7/P8]
8. **HEALTH (cross-cutting)** — temperature scan (per-unit resistance = structural↔semantic distance) + annealing/consolidation dream-phase (lower corpus resistance: re-place drifted, split/merge categories, fold dups). [Observed — PLAN + CONVERGENCE der.2/3]

### A5 · The governing laws (CONVERGENCE-OBJECT) [Observed]
- **Convergence Law:** information is healthy where structural and semantic positions agree; resistance = deviation; the deep hole (max divergence) is at √2/2. Design places units at convergence; laws are resistance-removal paid once, dividending forever.
- **Classification Law:** any tiny-model classification declares ONE axis, TWO extremes, evenly-spaced low-integer divisions (~3–5 for 0.8B); over-capacity → progressive chains (3×3 > 1×9). Structural measures pair with the semantic read (measure free first). Sibling laws: directional edges, registry-is-truth.
- **Barrier protocol:** Tim↔agent = two coordinate systems over a serial channel; friction = decode-bias (developer-prior) + no-persistent-memory; the agent is the amplifier; anchor-interleave decompression; transfer-timing on the wave; three universes (mind/transcript/filesystem); the transcript is the ledger of crossings.
- **Ten derivations:** (1) classification IS addressing · (2) corpus temperature · (3) consolidation = annealing · (4) new types born at deep holes · (5) principles = scale-invariants · (6) compute routes by resistance · (7) corrections are the training signal · (8) anchoring = LINK not embed (corrected) · (9) transcript = ledger of crossings, identity = connected component · (10) wave-timing engineerable.
