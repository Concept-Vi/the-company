The invariant application

There is one application. You never build a variant — not per device, not per viewer, not per mode. You write the invariant once, and it resolves itself against a coordinate. Everything you ever see is that one invariant, projected to where you are.

one invariant definition  ×  a context coordinate  —through→  the rules-resolver  =  the concrete surface, here, now

"Designed simultaneously" isn't extra work — portrait · landscape · desktop · native · spoken are just the resolver run at different points on one axis. Define once; the variants compute themselves.

the coordinate · its axes

Orthogonal, like the axes of a space — no order, no primary, nothing is number one. A surface is resolved against ALL of them at once (a point located by its full coordinate), never one-then-the-next. "Device" is the one you named — it's a spoke, not the hub.
