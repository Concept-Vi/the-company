## What this IS (the expansion)

**The Context Forager: the surface where Tim does his method at scale.** His method = curate context
→ think → AI produces. Today the curation step is trapped in his memory + a CLI. The forager is
SEARCH-AS-SCULPTING on a live canvas:

1. **SEARCH** across everything he's ever made/said — the corpus (1,400+ mined exchange-extracts ·
   584 repo files · transcripts · the vault · mockups · registries) — through the multi-projection
   substrate (Track-1 round 2: vectors=meaning · structure=files/registries · relations=links/tags ·
   time=seq/sessions). The facets he named: **time period · topic · project · principle · language
   (his vocabulary/terms)** — each facet is one projection queried.
2. **RESULTS AS CONNECTED CIRCLE NODES** on the canvas — not a list. Connections = the relations
   projection (shared session, shared address, semantic proximity, same project lineage).
3. **SCULPT THE SET:** whittle (remove nodes), grow (search again, add), iterate FAST — the canvas
   holds a WORKING SET, not search results. (The existing canvas already does nodes/selection;
   tldraw is the substrate; NodeShape/semantic-zoom is the pattern.)
4. **PROGRESSIVE DISCLOSURE per node** = the altitude grammar (Pulse: a circle + label · River:
   expand in place to see what it is · Nodes: full-screen open). Already the universal grammar.
5. **SELECT MULTIPLE → INTO THE BUILDER'S CONTEXT.** The S1 panel takes ONE indicated address today;
   the forager generalizes it to a SELECTION SET — the chosen nodes' content/addresses ride into the
   Claude Code session as the context block. **Then Tim thinks, and the builder produces.** The
   addresses make this exact ("Claude code know[s] [what I'm] clicking on, and where things are
   because of the addresses").

**Why it's the keystone for him:** "I have described and half made so many things... spread
everywhere." The half-made everything IS in the substrate already (mined, embedded, addressed) — the
forager is the missing READ surface for it. It is the UI of the resolver's retrieval face
(Track-1 round 2 §9.5: retrieval is a chain — vector shortlist → whittle → inject), with TIM as the
judgment pass.
