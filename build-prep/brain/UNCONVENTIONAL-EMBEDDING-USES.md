# Unconventional embedding uses — the ideas round (2026-06-11, conversation capture)

**Tim's ask:** "all my/our work is unconventional... neither of us know everything built or intended
... there is no consistency... no specifications, no accurate written plan... I know that it is
possible to resolve this, I just don't know how. Think about all the kinds of challenges that come
with my specific scenario and all of your ideas for resolving — heaps of ideas, different kinds."

## THE FRAMING INVERSION (load-bearing)
Conventional: code is truth, docs decay. HERE: **Tim's UTTERANCES are the truth; the code is the
decaying echo** (written by fragment-hearing sessions). The primary embedding axis is therefore
**utterance ↔ artifact**, not doc ↔ code.

## Kind 1 · Dissolving inconsistency
1. **Concept unification sweeps** — cluster everything by meaning; each cluster = one concept under
   N session-given names → proposed unifications (the built-twice radar at concept level).
2. **The synonym ledger** — the clusters' name-variant table as registry data; agents read it at
   boot; inconsistency becomes mapped terrain instead of debt.

## Kind 2 · Reuniting intent with implementation (the deepest vein)
3. **Provenance recovery** — per code unit: which conversation caused it? Attach to its address.
4. **THE PROMISE LEDGER** — per Tim-utterance-of-wanting: find its implementation; NO MATCH = an
   undelivered promise. The autonomous sweep that produces the undone-list nobody can remember.
   (Likely the single highest-value sweep available.)
5. **Intent triangulation** — embed all lossy echoes of one idea (mockups/transcripts/half-builds);
   the cluster CENTROID approximates the true intent; outliers = distortions. (His patterned-
   visibility method, mechanized.)
6. **Recurrence as vision-signature** — cross-space recurrence scoring separates the vision's
   skeleton (appears everywhere) from weather (appears once). (M3 corroboration generalized.)

## Kind 3 · Cold sessions stop being cold
7. **Boot-by-similarity** — a session declares its task in a sentence; the briefing assembles
   itself (nearest code/decisions/corrections/mockups). Boot = a query, not a reading list.
8. **Blast-radius by meaning** — entanglement beyond imports: conversations/rules/mockups touching
   the thing being changed. (The wire's scope-derivation gains a sense.)
9. **Pre-build collision detection** — mandatory "does this half-exist?" before any agent builds.
   (The incomplete-work law gets a SENSOR.)

## Kind 4 · Verification without specifications
10. **Done-ness as proximity** — built-thing vs its intent-cluster distance = a drift score standing
    in for spec-compliance.
11. **The foreignness detector** — style-embedding to find what doesn't sound like the Company
    (the ai_tics inversion at vector level).

## Kind 5 · Speaking at Tim's altitude
12. **Semantic addressing** — "the breathing circle thing" → ui://cognition/pulse; pointing by
    description; never canonical names. (Phone: describe → highlight.)
13. **The self-discovering toolbox** — agents find tools/flows/roles by need, not name (the MCP
    cold-agent problem dissolved by the same mechanism).

## Kind 6 · Structure that grows itself
14. **Auto-proposed relations** — affinity above threshold proposes links between addresses/records
    through the accept-gate. "Nothing is linked anywhere" → the graph assembles itself, Tim gardens.
15. **Idea genealogies** — one idea's embedding tracked across time-stamped exchanges: born, morphed,
    forked. The biography of each concept.

**The thread:** the embedding layer = the connective tissue replacing a long-tenured engineer's
familiarity — never leaves, covers everything at once. STRUCTURAL THREE: #4 promise ledger ·
#7 boot-by-similarity · #9 collision detection (they attack "no one knows everything built" head-on).
