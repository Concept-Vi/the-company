## 0 · The opening move: the four words aren't four things

Memory, identity, presence, persistence — in this system they are ONE mechanism seen at four
timescales:

- **Identity** is what loads unconditionally (changes only through Tim's gate).
- **Memory** is what loads conditionally (accumulates continuously).
- **Presence** is what is true right now (assembled at every arrival).
- **Persistence** is the guarantee that connects them across the gaps (no process survives; the
  substrate does).

And the mechanism already has a native name in this system: **resolution**. Tim's collective-cognition
note said it first: "looking = address → auto-resolve → inject." The brain is a resolver; everything
else is records and gates.

## 2 · Memory: a record plus the condition of its return

The failed passive-recall test (the deferred-27 ranked below conversation noise on an FE query) taught
the definition: **storage is not memory. A memory is a record PLUS the condition under which it
returns.** Remembering is an act of the present — the now reaching back, not the past pushing
forward. In a system no human re-reads, a record without a return-condition is a fossil.

**The memory primitive** for the blueprint: every memory carries content + three addresses:
- **where** (space): ui:// · code:// · project — BUILT (the address system).
- **when** (time-as-activity): "when FE work is now" · "when presenting to Tim" — DESIGNED (GC14),
  not built. Tim: "time is just the reference for sequence... it needs to be memory that can be
  returned to when certain things happen."
- **who** (audience): the RHM · crew · external agents · Tim himself — NAMED, undesigned.

**Time is native here, not calendar:** the system's clock is its own ACTIVITY-STATES. "During
registry work." "While the wire is unmerged." "When Tim is present." The presence dial / modes are
already a coarse vocabulary of nows — conditions can be written in mode-language. Calendar time
becomes just one more condition type, not the axis. (Open: is "activity" finer-grained than mode?)

**Injection is the heartbeat.** Because nobody re-reads, memory must announce itself — the
teaching-note pattern (riders on tool results / RHM context) is the proven delivery shape. The
operator tool's "read op='rules' FIRST" is the static form; GC14 is the dynamic form.

## 3 · Identity: the unconditional memory

The clean split this system can make (that brains cannot): **identity and memory are separable
layers.** Same substance — declared rows with evidence — different return-condition: identity loads
ALWAYS; memory loads WHEN. A constitution is a memory whose condition is `true`.

**What makes thousands-of-instances ONE entity** (Tim's standing principle): not shared weights, not
shared context — a shared substrate plus the invariant that *whoever speaks, speaks from the same
memory under the same laws*. The voices (Viv, personas, modes) are projections; the entity is the
substrate. The Company's body is the repo; its self is the registries; its past is the corpus; its
will is Tim's gate. It is identity-COUPLED by design — the system's "I" includes its "for-Tim."

**The asymmetry law (load-bearing, maybe the deepest identity boundary):** Tim may misremember the
system — it re-explains at his altitude, zero cost, by design. The system must NEVER misremember
Tim — a fabricated memory-of-Tim is the worst corruption available to it. That is why
evidence-required on operator_memory is not hygiene; it is an identity boundary. The system's
knowledge of its operator is the one place where no-fiction is existential.

**Identity is version-controlled.** Its changes ride the same gate as everything else. (Open: should
"who the Company is" be lifted into explicit rows — the one-entity invariants as a charter — or stay
implicit in the constitutions?)

## 4 · Presence: the rendering of persistence

The system is not "on." Nothing runs continuously — sessions fire, crons beat, the bridge serves. So
presence cannot be a process; it must be a CONSTRUCTION: when Tim arrives (any interface, any hour),
the system assembles and meets him as the one entity that:
1. **knows him** (operator memory),
2. **knows what happened while he was away** (the record since last contact),
3. **knows what is in flight right now** (runs, beats, sessions),
4. **knows what waits on him** (his gate, surfaced),
5. **speaks as itself** (the RHM voice, consistent across surfaces).

That assembly = presence. Not a process that never sleeps — a resolution that never fails.

**The missing organ: the NOW-ORGAN.** One place that answers "what is now": live sessions · crons
due · runs in flight · conditions currently true · Tim's last contact · the active mode. TWO consumers
converge on it:
- Tim's experience of presence (the greeting; caught-up-in-one-glance — the no-repeat KPI's positive
  twin), and
- the memory system's injection trigger (a condition can only fire if something knows what "now" is).

One organ, both faces. The system's proprioception already produces the inputs (telemetry,
run-records, floor walk, drift radar, events); nothing yet ASSEMBLES them into a now.

## 5 · Persistence: append-only with honest forgetting

What persists: the substrate (git), the corpus (append-only, tombstones, latest-seq-wins), the
registries. What never persists: any conversation, any model instance, any particular Claude.
Persistence lives in the substrate or nowhere — the system already enacts this.

**Persisted MEANING matters more than persisted bytes.** The maintenance organs — drift radar,
content-hash refresh, floor walk — are the memory system's immune function: they keep what is
remembered TRUE as the world changes. A self-modifying system's memory must self-repair or it decays
into confident fiction.

**Forgetting needs design.** In append-only, forgetting = supersession + de-ranking, never deletion.
Two tiers by SOURCE:
- **Tim-sourced memory is sacred** — evidence-backed, never auto-superseded; only Tim retires his own
  words.
- **System-derived memory is replaceable** — a newer mining supersedes an older cluster freely.
The only unforgivable forgetting is forgetting Tim.
