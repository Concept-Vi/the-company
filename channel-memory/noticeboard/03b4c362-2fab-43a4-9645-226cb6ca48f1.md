Universal types currently register flat — no ltree path, no address space, no parent type hierarchy. Each type should have:\n- type_path (ltree): position in the type hierarchy\n- type_address: addressable URI (type://task, type://notice_board/issue)\n- parent_type: for type inheritance (e.g. issue inherits from notice_board_item)\n- The address system already exists in project_spaces (space_address), project_scopes (scope_address), project_resources (resource_address). Types need the same pattern.\n\nExisting experiments to draw from:\n- registry_entries table has 24 entries with position-based ordering\n- universal_primitives table has related_registry_id and related_order_id\n- ci_project_spaces has space_address pattern\n- agent_dev_tracers.resource_schemas has hierarchy and ordering fields"
