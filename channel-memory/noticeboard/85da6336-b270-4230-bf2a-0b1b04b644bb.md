Tim confirmed SMS notifications ARE firing on approval proposals — the event mesh works. But the flow is incomplete:\n\n1. TEMPLATE: Needs summary of what's being approved, tap-to-open link to project space, yes/no reply instructions\n2. REPLY HANDLING: When Tim texts 'yes', route through SMS inbound → event_routing_registry → approve_proposal RPC. Chain exists in pieces.\n3. CONFIRMATION: After approval executes, send back confirmation SMS (e.g. 'Approved: @task dispatched to Keeper')\n4. NOTIFICATION PREFERENCES: Configurable per-project in project space settings — which channels, what triggers, what detail level\n\nSame universal mechanic: Registration (notification_preferences) → Resolution (channel + template + content per event type) → Trigger (proposal/status/completion events) → Connection (SMS delivered, response routed back)\n\nInfrastructure exists: sms_messages, sms_threads, event_routing_registry has SMS types, send_notification capability at L3. Gap is approval-specific template and reply-to-approve routing.
