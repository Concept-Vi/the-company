## Design Proposal: Mobile Product Improvements via Templates, Tokens & Context Modifiers

**Objective:** Deliver an app-like mobile experience without front-end code changes — leveraging the existing composition infrastructure (context modifiers, theme tokens, templates, behavior bindings).

### Current State Analysis
From the operational gap trace and product vision context, the mobile experience currently:
- Uses hardcoded @media breakpoints (768px) instead of spatial context modifiers
- Lacks touch-optimized interaction patterns (swipe, bottom sheets, haptic feedback)
- Has visual hierarchy issues on small screens (56px touch targets not enforced, inadequate safe padding)
- Missing narrative context adaptation — prestige/branded experience not triggered by spatial/temporal modifiers

### 3 Highest-Impact Improvements (No Front-End Code Required)

#### 1. **Spatial-Aware Touch System via Context Modifiers**
**Problem:** Touch targets are inconsistent. The 56px minimum touch target token exists but isn't enforced.

**Composition Solution:**
- Add/modify `context_modifier` entries with `context_dimension='spatial'` and `context_value='mobile_phone'`
- Token overrides: `sys-mobile-min-touch-target: 56px`, `sys-mobile-safe-padding-top: 44px`, `sys-mobile-safe-padding-bottom: 34px`
- Template update: Add `data-spatial="mobile"` attribute to block templates for touch-optimized rendering
- Behavior binding: Add `touch_feedback` animation on `trigger_event='tap'` for mobile spatial context

**What Users DO:** Tap buttons and cards with confidence — clear visual feedback, no mis-taps, thumb-friendly zones.

**Files to Touch:** `context_modifiers` table (add mobile variants), component templates, behavior_bindings (add tap feedback)

---

#### 2. **Prestige Bottom Sheet Navigation Pattern**
**Problem:** Project navigation requires full page transitions. No modal/overlays for quick context switching.

**Composition Solution:**
- Create new `bottom_sheet` component type via `add_component` (template + theme specs)
- Context modifier for `mood_tone='focused'` triggers bottom sheet expansion for "quick actions"
- Token cascade: `sem-elevation-modal: 24`, `sem-radius-lg: 16px` for premium feel
- Template structure: `bottom_sheet` → `bottom_sheet_header` + `bottom_sheet_content` + `bottom_sheet_handle`
- Behavior binding: `swipe_down` triggers `close` action; `drag_handle` interaction

**What Users DO:** Swipe up from project cards to see scope list, swipe down to dismiss — app-like navigation without page loads.

**Files to Touch:** New component registration, behavior_bindings for swipe gestures, context_modifiers for mood→component mapping

---

#### 3. **Adaptive Visual Hierarchy via Temporal + Spatial Context**
**Problem:** Visual hierarchy doesn't adapt when user context changes (late night, quick check, deep work).

**Composition Solution:**
- Context modifiers for `temporal='evening'` + `spatial='mobile'` combine to trigger "focus mode"
- Theme cascade: Darker palette, reduced decorative elements, larger text (`ref-font-size-base` scaling)
- Template: Create `focus_card` variant of `card` component with simplified layout
- Accessibility spec: High contrast mode automatically enabled for evening temporal context
- Animation: Subdued transitions (`duration_fast: 150ms` instead of 300ms)

**What Users DO:** Open the app at 11 PM — interface automatically simplifies, reduces cognitive load, emphasizes critical alerts and approvals.

**Files to Touch:** context_modifiers (temporal rules), component_theme_specs (focus variants), accessibility_specs (auto high-contrast)
