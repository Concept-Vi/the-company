**Why each piece reuses a real seam (not invented):**
- **Scope** = the `frame` grammar already shared across `decision://`, `vi-vision://`, `image://` (`parse_decision_address` etc.). Tim's "same mechanism, just what it attaches to" *is literally the frame*. Don't build a new scope system.
- **Freshness** = `@v<n>` (`split_version`/`versioned_address`) + `Provenance.content_hash`. Stale-detection is *free*: compare the pinned `captured_at` hash to the target's current hash. The brief lists "versioning of the target so a stale envelope is detectable" as an addition — it's already a primitive, just wire it.
- **Captured content** = `cas://` immutable snapshot. This is the difference between "the agent re-reads possibly-changed content" and "the agent sees exactly what Tim saw." Pin it.
- **Editable location (UI mode)** = the **real** `ui:// → code://` resolver (`Suite.resolve_scope`, reading `design/_system/addresses.json` + `code-symbols.json`). This is verified to exist. The envelope should carry the *resolved* `code://` + deps so the member gets a direct edit target — Tim's "reduce the agent's effort" goal, fulfilled by a join that already runs.
- **Lifecycle** = a new `item_types/annotation.py` row with registry-declared states. "Submit-all" = file N annotation items at once + post the routing edges. Annotation-as-item-type drops cleanly into the board.
- **Intent** = a registry ref (a new `intent_types/` or reuse a mark direction). Don't hardcode the enum.
