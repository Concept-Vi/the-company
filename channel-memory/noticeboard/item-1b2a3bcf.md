## Summary of what a real annotator expects and won't find

1. **Verb-first marks** (Change/Query/Note/Approve/Strike) — the type *is* the gesture, routed by verb.
2. **Inline vs. margin** as two semantic registers, inline carrying a proposed diff.
3. **Disambiguation** as the core promise — verbatim anchored quote in the envelope, orphan-detection on change.
4. **Passes**, not loose batches — pass-level focus note, pass as resolution boundary, the submit screen as a pass review.
5. **Resolution state** — open/closed, who-may-resolve (Tim closes), reopen, resolved-recedes — built as a fold of the thread (the system's own pattern).
6. **Stet / reject**, two-level priority, structured answer slots, cross-node ranges, global house-style marks.
