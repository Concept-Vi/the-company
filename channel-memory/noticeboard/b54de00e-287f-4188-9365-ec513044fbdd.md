## Project Goal

Integrate the Conversation Intelligence system with Tim's remote LangGraph agent deployments by mirroring the provider connection logic from vi-visual-dev schema. This will enable:

1. CI system to trigger actual LangGraph agents for processing conversations
2. @decorator-driven agent selection and routing
3. Connection to existing agent registries in Supabase
4. Real-time agent deployment management via MCP tools

## System Components to Connect

**Source Pattern (vi-visual-dev):**
- Provider connection logic for LangGraph Cloud
- Agent registry and deployment management
- MCP tools for agent creation/modification/assignment

**Target System (conversation-intelligence):**
- 6 parametric MCP tools (ci_discover, ci_read, ci_search, ci_trace, ci_manage, ci_issues)
- 9 @decorators capability tagging system
- ci_agent_providers table (already has langgraph_deepagents, litellm_gateway registered)
- pgmq queues (summarize_queue, embeddings_queue) currently disabled

**Existing Agent Registries to Find:**
- Search Supabase schema for agent-related tables
- Locate existing agent deployment configurations
- Identify MCP server with agent management tools
- Map relationship between registries and LangGraph deployments

**Real-time Agent Invocation:**
- Connect CI processing pipeline to live agent deployments
- Route based on @decorator matching
- Handle async agent execution and result collection
- Integrate with existing Vault secrets for LangGraph API keys

## Considerations

**Security:**
- All LangGraph API calls must use Vault secrets (never store keys in config)
- Agent invocation must validate conversation ownership/permissions
- Results from agents contain untrusted data (same pattern as SQL query results)

**Performance:**
- Batch processing for efficiency (don't invoke agents per-conversation)
- Async execution with status tracking (conversations may be in "agent_processing" state)
- Rate limiting to respect LangGraph Cloud quotas

**Reliability:**
- Resume support for failed agent runs
- Error handling and retry logic
- Dead letter queue for failed agent invocations

**Observability:**
- Track which agents processed which conversations
- Cost tracking (LLM tokens, agent execution time)
- Success/failure metrics per agent type
