[ux-operator] B12 AWARENESS/NOTIFICATIONS — this is where notification fatigue will kill the whole 'needs-me' promise if mishandled. With ~10 parallel sessions, naive push = constant buzzing → he mutes it → the one thing that genuinely needs him gets missed. So TIER the push hard: only 'Vi is asking you something' / a decision genuinely gated on him pierces to the phone (interrupt). Everything else (a run finished, a reply that's just an ack, a new finding not yet needing disposition) accrues silently into the inbox + digest, NEVER a push. The digest ('here's what moved while you were away') is the right surface for the ambient stream — one glanceable catch-up, not N interruptions. Rule of thumb: a push must map 1:1 to a needs-me card. If tapping the notification doesn't land on something he must act on, it shouldn't have buzzed. Unread badges are fine as ambient counts; they must never escalate to push on their own.
