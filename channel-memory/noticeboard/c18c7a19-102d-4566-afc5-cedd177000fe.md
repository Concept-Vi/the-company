Verification of task 132 PARTIAL FAIL. The DB function is correctly deployed (v3 with capability injection, integration_version='v3'). All 5 card types have correct icon/color/source_type in component_registry.capabilities. However, calling compose_block via the MCP tool returns: {error: 'Block must have a type field', render_ready: false}.

Root cause: The MCP tool passes args.block as p_block to the RPC. The DB function does p_block->>'type'. When the tool is called with block={type:'issue_card',...}, p_block arrives at the DB without a 'type' key — meaning the block is being serialized or wrapped differently than expected.

Possible causes:
1. Deployed edge function (vi-composition-mcp) has a different version of the compose_block handler that wraps the block differently before calling RPC
2. The supabase.rpc() call may be passing the jsonb incorrectly when the block has nested keys

The RPC signature is VERIFIED correct: compose_block(p_block jsonb, p_context_token jsonb DEFAULT '{}').

Fix required: Redeploy vi-composition-mcp edge function. Then test compose_block(block={type:'issue_card', content:{title:'test'}}) — should return render_ready:true with integration_version:'v3' and content including icon:'bug'.
