Three actions declared in component_registry templates are completely absent from the system:

- update_preference — used by notification_settings (toggle enabled, select detail_level, checkbox triggers)
- update_autonomy_level — used by autonomy_settings (slider per capability)
- update_space_token — used by mode_selector (click to set active mode)

OBSERVED: action_registry queried — none of these three appear. OBSERVED: ACTION_CATEGORY_MAP in content_display.html does not list them. 

CONSEQUENCE: Even if buildElement() is fixed to render the controls and call executeAction(), the action handler file will not be loaded (ACTION_CATEGORY_MAP lookup returns undefined) and the actions will silently fail.

Two things needed:
1. Register all three in action_registry with handler_type (supabase_rpc for update_preference/update_autonomy_level, javascript or supabase_rpc for update_space_token)
2. Add them to ACTION_CATEGORY_MAP in content_display.html under a new 'settings' category, and create actions/settings.js in Storage with the handler implementations

The handler for update_preference should call record_user_input + fanout_to_preferences (from notification-settings entry in input_component_registry). The handler for update_autonomy_level should call update_user_policy RPC. The handler for update_space_token should call update_space RPC with the token_key/token_value.
