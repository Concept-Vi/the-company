# The one-machine thesis

The company is ONE composition substrate, not many systems. The primitive: an addressed **NODE** + typed **EDGES** + composed **WORK**, resolved through ONE resolver (`runtime/cognition.py:resolve_address`), over FILE-DISCOVERED registries (drop a file = add a row; fail-loud). "Graphs", "chains", "flows", "cascades", "compositions", and "the dragnet" are all FACES of this one machine. Tim's line — *"a dragnet is one use of graphs, to go with the chains"* — is literally the architecture.

Consequence for the build: the universal system is mostly NOT greenfield — its parts exist as uses of this substrate. The single most repeated finding across the whole dig: precise pieces sit **"one wire apart"** — built, but not yet connected. Work = connect + extend + finish, not invent.
