**For a non-developer user opening this project space on their phone for the first time:**

## Priority 1: Fix Mobile Scrolling — Keeper Blocks Content (CRITICAL)
**What they NOTICE:** They can only see the top of the page — swiping does nothing, content is trapped behind a big gray panel.
**What they try to DO:** Scroll to see their project items, but the Keeper chat is stuck in the way.
**Problem:** The Keeper takes 40% of viewport and content below the fold is unreachable. The user sees ~2 items max.
**Fix:** Collapse Keeper to a floating bubble on mobile; expand on tap only.

## Priority 2: Fix Board View — Complete Layout Breakdown (CRITICAL)
**What they NOTICE:** The Board view shows an empty "ACTIVE" column and nothing else — their 53 OPEN items are missing.
**What they try to DO:** Find their tasks, swipe to see other columns, or tap something to navigate.
**Problem:** All columns except the first are off-screen or hidden behind the Keeper. "Right-click to move" appears on a touch device (impossible).
**Fix:** Implement swipeable single-column board view or column picker dropdown. Remove right-click instruction on mobile.

## Priority 3: Replace Developer Language with User Language (HIGH)
**What they NOTICE:** Everything is gibberish — "compose_block", "cascade_registry", "RPC invocation" — they cannot understand anything.
**What they try to DO:** Read what the system did, understand what needs their attention.
**Problem:** Content displays technical implementation details instead of user-facing descriptions.
**Fix:** Add a user-readable layer that translates technical content to actionable plain English in an expandable section.

## Priority 4: Add Basic Actions — Interface is View-Only (CRITICAL)
**What they NOTICE:** They can see items but cannot tap anything to change it, create new, set priority, or assign.
**What they try to DO:** Create a new issue, edit a task, change status, add a comment, move a card.
**Problem:** The entire interface is read-only. Only Approve (on raw JSON) exists. No creation, editing, status change affordances.
**Fix:** Add + buttons on each view, status change dropdowns on cards, inline edit on tap.

## Priority 5: Add Search — 551 Items with No Way to Find (HIGH)
**What they NOTICE:** They know something exists but cannot locate it. The Cmd+K button is either non-functional or unstyled.
**What they try to DO:** Find a specific item, search by name, filter results.
**Problem:** No search bar, no working command palette, no in-view filtering beyond basic type buttons.
**Fix:** Add a persistent search bar with full-text search across notice board, resources, and content. Wire Cmd+K or remove it.

---

- Loading states show generic spinner (not branded)
- Error pages show raw technical messages
- No dark mode support
- Stats render as stacked text (wasted vertical space)
- Members show "User owner" with no names or avatars
