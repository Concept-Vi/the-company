The Notice Board filter bar shows type labels like "cognitive_guide", "diagnosis", "observation" as raw snake_case strings. These are internal type names, not user-facing labels. "cognitive_guide" should be "Guide", "observation" should be "Note" or "Observation" with an icon. All filter labels need to use the display_name from the type registry, not the internal type_name. Each filter should have an icon and a human-readable label.
