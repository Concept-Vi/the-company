Blocked by #255 (provider unification research). Three interconnected design surfaces need to be worked through the project space system, not built from a single agent's narrow view: (1) Session persistence — how agent identity resolves at session start, how session state connects to the graph, how thread continuity works across the 5 existing thread systems. (2) Agent inbox — how tasks, proposals, and dispatched work arrive at the right agent through decorators and the autonomy ladder, how the inbox connects to the notice board and approval chain. (3) Dispatch-cascade integration — how decorators bend the task graph, how autonomy levels gate what agents can do vs must propose, how types cascade through agent registrations, how block compositions render agent-created content. All of these touch provider configuration, model resolution, token cascades, graph edges, and the event mesh. Building any piece without the unified view introduces the same drift that scattered the system in the first place. The project space IS the design surface — use notice board, resources, graph, and the Keeper to work through this collaboratively.
