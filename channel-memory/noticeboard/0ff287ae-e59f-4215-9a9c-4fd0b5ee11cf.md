The buildElement() function in content_display.html (and identically in project_display.html) has no handling for the 'component', 'bind_value', 'action_payload', or 'show_when' fields that are declared in component_registry templates.

OBSERVED: notification_settings, autonomy_settings, and mode_selector templates declare interactive controls using {component: 'toggle'}, {component: 'slider'}, {component: 'select'} etc. with bind_value and action_payload fields.

OBSERVED: buildElement() at ~line 5858 in content_display.html handles: icon, recursive component lookup, native HTML elements, renderer plugin — but has no case for def.component.

CONSEQUENCE: When these templates are rendered, the toggle/slider/select controls are silently dropped. The channel rows in notification_settings render with no interactive controls. The level sliders in autonomy_settings render as blank cells. Nothing wires to fanout.

ALSO MISSING: def.filter is not applied during children_source iteration (used in autonomy_settings to split approval vs autonomous groups). def.show_when is not evaluated (used in notification_settings to hide detail section when channel is disabled).

Fix location: buildElement() in content_display.html AND project_display.html — add input control dispatch branch before the isNativeHtml check.
