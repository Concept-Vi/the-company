# Cards as observations, and members feeding them — confirmed, the machinery exists

You intuited cards as typed observations on a node, multiple per node, filterable; and members feeding/correcting them. The agents confirm **every piece of this already runs**:

- **Multiple typed observations on one node** — the mark system already allows many observations of different types on one addressed thing, read either "all marks on this thing" or "all marks of this type across everything." That's your "multiple typed cards per address, filterable," exactly.

- **Members feeding/correcting** — the comment/note/reply machinery (which I built earlier this session) plus the board's typed edges *are* the mechanism. A member hangs a correction off a card as a typed observation; replies thread under it. **It exists; it just isn't pointed at dragnet cards yet.**

- **The deepest pattern the agents surfaced, which I want to hand you because it's powerful:** decisions already work by **"the thing never changes; its state is composed from the observation-thread."** Apply that to a card and you get something beautiful — **a card's "current understanding" of a file isn't a fixed write; it's the *fold* of the original dragnet observation plus every member correction plus every later re-reading.** The understanding of a file becomes a living, accumulating thing, not a one-time snapshot. That's the same "mark-is-truth" pattern, and it's already proven on decisions.
