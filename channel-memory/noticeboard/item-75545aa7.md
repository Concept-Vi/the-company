# The failure modes that must be structurally prevented (problems, not a solution)

Whatever architecture resolves this must structurally prevent the ways interfaces rot — these are the problems to design AGAINST:
- DUPLICATION — the same thing built twice in different places.
- DRIFT — code and documentation diverging.
- INCONSISTENCY — pieces that don't obey a shared contract; half-broken states shipped silently.
- MONOLITH ROT — one file accreting special cases until it's unmaintainable.
- THE FAST-HACK TEMPTATION — that the quick way and the correct way are different (they must become the same).
