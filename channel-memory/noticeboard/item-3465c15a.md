## 2. THE biggest reuse asset — the company repo is ALREADY a vault (on paper)

`/home/tim/company/docs/vault-conventions.md` (alias **"Vault Conventions"**, tagged `obsidian`) is the canonical spec saying:

> "how `~/company` is **simultaneously a code repository and an Obsidian vault** ... the documentation *is* the knowledge graph."

It defines a complete dual-repo convention already in force across the repo:
- **Every folder has an `AGENTS.md` constitution-note** with YAML frontmatter (`type/module/aliases/tags/governs/relates-to/status`).
- **`[[wikilinks]]` carry relations** between constitutions (via unique `aliases`, since every file is literally named `AGENTS.md`).
- **`MAP.md`** (alias `Company Map`) is the declared vault home with a rendered relationship graph + auto-maintained REGISTRY block.
- A **two-vault boundary** is already articulated: `~/company` = built *reality*, `build-prep/` in a counterpart vault = design *intent*; they cross-reference **by prose name** because "Obsidian does not resolve links across vaults."
- Freshness is owned by a **Coherence Gate** (commit-time check).

`AGENTS.md` line 58 and `MAP.md` line 15 both already state "This repo is **also an Obsidian vault**."

**This is the spine the host-idea plugs into.** The convention layer, the link grammar, the home note, the freshness-gate concept — all already designed. The board-as-vault idea inherits a mostly-built knowledge-face convention.
