### ❌ Image & file attachment (BD-H)
blob:// exists, so it *feels* free, but mobile pickers, MIME handling, thumbnails, draw-on-image are not free, and they're not load-bearing for the cycle. The one attachment that matters — a screenshot for UI feedback — Tim can do via the photo library into a text comment, or just describe it. Defer the in-app attachment flow until the loop is humming and a real need surfaces. Voice-note attachment is even further out (real, since he prefers voice, but additive — see below).
