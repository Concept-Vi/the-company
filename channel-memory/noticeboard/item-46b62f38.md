## 10 · The cleanest path (recommendation, held tentatively)

Not a single answer — a **staged spine**, each stage independently valuable and reversible:

1. **L0/L1, A2 (mirror), read-only.** One-way sync `noticeboard/ → vault/`. Open it in Obsidian (reuse the existing `.obsidian`). Immediately: search, Properties, Bases board-views, iOS app + Sync on the phone. **Zero code touching the canonical store.** This alone delivers "Obsidian as a second lens on the phone" — which is the headline ask — with no risk to the fabric.
2. **L2 graph shim, into the mirror.** A generator reads each item's `links:` and `order:`, and writes (a) body inline-fields `kind:: [[target]]` for `board://` edges, (b) `![[…]]`-sequence assembled-document notes, (c) `aliases:`/title display. Now graph view, backlinks, and the assembled-document reading view light up — **on the mirror, canonical files untouched.**
3. **Decide cross-scheme stubs (§7)** — start "leave as text," add image-embed stubs when the image work lands.
4. **Writeback only as `inbox/`→`file_item` ingestion (§5)** if/when an Obsidian-side input path is wanted — never in-place mutation.

**Why mirror-first over point-straight:** it makes "host not master" *structural* (a one-way sync can't promote Obsidian to a writer by accident), it's the right shape for the mobile/iOS goal, and it keeps the git-tracked company repo free of `.obsidian` churn. The cost is one sync component — cheap relative to what it de-risks.

**The contrarian case for point-straight (kept alive):** if the goal is the *truest* single-source-of-truth and a fast demo, A1+L0 is "open the folder, done" with literally zero new code. It's the better choice for a quick *look* before committing to the mirror plumbing. The grid (§1) means we don't have to choose globally — we can A1 for an exploratory look today and A2 for the durable surface.
