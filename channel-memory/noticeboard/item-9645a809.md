# Your scopes idea is already built — just not yet pointed at dragnet outputs

You said: *"project private, but there would be multiple things… attached to a channel… a one-off for an individual session… connected to the project the channel is part of… put into global… attached to a working directory… it's the same mechanism really, it's just what a thing attaches to."*

The agents found this **already implemented and working** — for decisions and for the visual-component factory. Here's the real shape:

- There's a scope ladder that already exists: **everyone's (global) · this project's · this user's · this session's.** It has a **rank**, and a **most-specific-wins cascade**: when something is looked up at session-level, the system returns the session-private version if it exists, and otherwise *falls back outward* — session → project → global. So a session can have its own private view that shadows the shared one, automatically.

- And — this is your exact insight, confirmed mechanically — **the scope is carried *in the address* as a "frame."** A decision's address is `decision://‹whose›/‹id›` where *‹whose›* is the scope. **Scope literally is "what it attaches to," encoded in the address.** You said "it's the same mechanism, just what a thing attaches to" — the system implements scope as *exactly that*: a frame in the address naming the owner.

So your six scopes (channel, session, project-of-the-channel, global, working-dir, "heaps") are **one mechanism that already exists** — it just hasn't been extended to dragnet outputs and channels yet. The ladder is built; we'd be adding "channel" and "working-dir" as new rungs and pointing dragnet cards at the right rung.

**And the self-resolution principle you remembered is real and disciplined** — richer than I knew. A session finds its *own* identity through an injected self-marker, never by typing an ID, and the system **fails loud if it can't be sure which session is "self"** (it refuses to guess, because mis-identifying self would corrupt memory). That's not just convenience — it's a safety property: **you can only ever act as yourself, so you can't accidentally (or maliciously) act as another.** Your instinct to generalize "self" to "here" (this channel) and "this project" is exactly the relative-root idea, and the foundation for it is already laid (the system already anchors things to a stable root rather than wherever-you-happen-to-be).
