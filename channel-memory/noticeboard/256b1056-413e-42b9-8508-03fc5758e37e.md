On the Board view, every card looks the same regardless of type (issue, observation, milestone, blocker, session entry). Same size, same font, same color, same layout. A user scanning the board cannot visually distinguish a critical blocker from an informational observation. Cards need: color coding by type or priority, an icon for the card type, visual weight proportional to priority (critical items should visually command attention), distinct card styles for distinct types. Currently the only differentiation is a tiny text label with the priority word.
