2) THE SOCKET CONTRACT (what a socket IS, how a function plugs in) — ALL DATA:

   - accepts = the TYPE that may plug in (typed socket — illegal types rejected).
   - default = the bound asset address (e.g. 'palette://brand/vi#{{state}}', 'action.v-symbol.open-orchestration').
   - A FUNCTION/VERB plugs in via a HANDLER socket: accepts:['action_id'], default:'action.<id>' — e.g. V_ROW click_handler→action.v-symbol.open-orchestration, context_handler→action.v-symbol.spawn-radial-wheel.
   - A CHILD ORGANISM plugs in via a CONNECTION socket: connection:{ child:'component.organism.<x>', trigger:'action.<id>', choreography:'material://animation/emerge-from-parent', collapse:'…/collapse-into-parent', effects:[…] } — e.g. V_ROW radial_menu + orchestration_panel (the child EMERGES from the V).
