The detail modal (openNoticeDetail) is built with template literals and hardcoded HTML. It should be a notice_detail component in component_registry with: (1) Default template: modal with type accent header bar, icon, title, status/priority badges, markdown description, related items, action footer (Resolve/Close/Reopen). (2) Mobile variant: full-screen sheet with type color header, large icon, swipeable actions, fixed bottom action bar with safe area padding. (3) Resource detail variant: similar layout but for project resources with content sections, tags, decorators. The detail view is where users DO things — change status, add comments, see connections. It needs to be the highest quality composed surface. Currently works but looks generic.
