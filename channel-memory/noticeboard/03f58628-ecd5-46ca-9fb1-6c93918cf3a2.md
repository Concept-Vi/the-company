fanout_to_user_inputs function (public schema) does:\n\nINSERT INTO content_block_events (event_type, user_id, content_id, event_data, correlation_id)\n\nBut content_block_events has no event_data column. Actual column is block_data.\n\nThis means any user_inputs fanout path that hits fanout_to_user_inputs will throw a column-does-not-exist error. Input capture events for content_block_events are silently broken.\n\nFix: CREATE OR REPLACE FUNCTION public.fanout_to_user_inputs(...) — change event_data to block_data in the INSERT column list.\n\nFix class: column rename. Not a missing table, not a missing function — single column name mismatch between function definition and table schema.\n\nRelated: VERIFY #135 (migration batch integrity check where this was found).
