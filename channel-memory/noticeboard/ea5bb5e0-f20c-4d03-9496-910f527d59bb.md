When updating hosted_content with compose_project_page output (156KB, 13 blocks), handle_content_block_added_trigger tries to pg_notify with last_block in payload. pg_notify has 8KB limit. Error: payload string too long. Workaround: insert blocks in batches or truncate the notification payload. Root fix: the trigger should send block_id + content_id only, not full block content. Architectural class: notification-payload-assumes-small-blocks — trigger designed for single-block additions, not bulk content population.
