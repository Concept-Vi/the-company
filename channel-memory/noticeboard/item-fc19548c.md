# Breakdown · Mobile / iOS usability (a first-class constraint)

SPECIFIED: Everything is on his iPhone. Deep consideration for real usability. Convenient. Non-obtrusive — present when needed, not always. Small screen. He is not a product designer.

IMPLIED: Thumb-zone design; minimal chrome; progressive disclosure; gesture-first with button fallback; performance on mobile Safari; installable PWA (full-screen, home-screen icon); offline tolerance + sync; large tap targets; haptics; respect iOS scroll/selection conventions; readable typography + good contrast; light/dark. Crucially: much of this can only be truly VERIFIED on his actual device — that verification is a needs-Tim loop, never assumed-green.

ADDITIONS: PWA install; haptic feedback on level changes + submit; voice input for the comment text; swipe gestures (e.g., swipe a draft to delete); reading-progress; remember scroll position per document; a one-handed mode.
