project_resources.created_by is text NOT NULL — designed for context token identifiers like 'user:ebe5f9c7' or 'agent:vi-composition-mcp'. Current MCP tools don't have user context (no login), so they default to agent identity. In v2 with auth, created_by resolves from the authenticated session. This is the same pattern as actor_id on hosted_content and correlation_id on intents — provenance tracking through the system.
