## What I'd actually trust to ship vs. not

**Could be real soon (it mostly exists):** tap-a-block → composer → posts a `commented_on` edge → lead replies in place. That loop *runs today* (`doc_review_server.py` + `cc_board`). The draft queue + submit-all + an inbox overlay is an honest, bounded extension of it. MSG2's concrete UI fixes (one-line top bar, icon buttons, PWA theme-color, kill the redundant selected-text echo, send-now button) are real and small. **Build outward from this.**

**Do NOT pretend is near:** the semantic document tree (#1), the slide-to-scale gesture as a requirement (#2), live element→source resolution (#3), member-gated data access + channel convergence (#4), self-resolving editable locations (#5), durable anchoring across edits (#6). Each is presented as "finishing what exists." Each is a build. Sequenced naively, #1/#4/#6 are the ones that strand the repo.
