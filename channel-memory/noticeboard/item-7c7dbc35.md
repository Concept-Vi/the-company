# Where this is heading toward actionable (grounded, not a decision for you)

Because the parts exist, the first build gets concrete and small instead of vast:

1. **Make dragnet cards into observations on file-nodes** — instead of standalone records, each card becomes a typed observation hung on its file's address, using the mark machinery that already runs. (The file's content-signature, already recorded, gives you free freshness — the node knows when its file changed.)

2. **Make the receipt the hinge** — the run becomes the node that its cards hang off, so "open channel → see its dragnet runs → walk into the cards → open a file" is one path along edges that already traverse.

3. **Scope the outputs by frame** — reuse the existing scope ladder so a dragnet's cards can be session-private, channel-shared, project-shared, or global, by *which frame they attach to* — your exact "same mechanism, just what it attaches to."

4. **Point the comment/edge machinery at the cards** — members feed and correct them through the annotation layer that already exists.

Each step is *connecting things that already run*, not building from zero — which is why your intuition keeps landing on real seams. The bigger horizon (the lenses, vision, the one unified walkable graph) then attaches to this same spine as more observation-types and more edge-types.

**We are not deciding any of this yet** — you're still building the picture, and that's right. The point of laying it out this way is so you can feel whether the shape matches what's in your head, now that you can see it's *grounded* rather than imagined.
