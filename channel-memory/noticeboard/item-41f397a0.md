# Breakdown · The block hierarchy & the long-press gesture-selection

SPECIFIED: Blocks form a hierarchy (word < sentence < paragraph < under-a-heading < whole). A LONG-PRESS activates selection (distinct from a normal tap). While holding, move thumb UP = coarser scale, DOWN = finer; release = commit that level. Highlight = word. Selection starts at the lowest granularity at the tap point. "Space/zone" regions (after a paragraph, between bullets) are themselves selectable. Left/right reserved (empty slots for later). The same gesture should adapt to UI mode.

IMPLIED: There must be a PARSED HIERARCHICAL MODEL (a tree) of the content so "one level up/down" is well-defined at ANY point and any format — not flat text. A tap coordinate must map to the deepest node there, then walk ancestors for up/down. Needs LIVE visual feedback of the current level during the hold, and iOS HAPTIC ticks on each level change. The gesture must not fight scroll or native text selection. "Zones" implies even non-text gaps are addressable structural nodes. Needs a non-gesture fallback (buttons) for accessibility / when the gesture is awkward. The hierarchy is TYPE-AWARE (a heading's "up" = its whole subtree; a list item's "up" = the whole list; a sentence's "up" = its paragraph).

ADDITIONS: A persistent small "level" indicator while selecting; snapping + a preview outline of exactly what will be committed; multi-block select (lasso several for one comment); remembering a preferred default level; pinch to grow/shrink selection; left/right could map to "select previous/next sibling" or "extend selection to include the next block.
