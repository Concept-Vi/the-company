OBSERVED: content_block_events has column block_data (jsonb), NOT event_data. Five functions were inserting with the wrong column name and would throw ERROR: column "event_data" of relation "content_block_events" does not exist at runtime.

Affected functions: fanout_to_calendar, fanout_to_kanban, fanout_to_user_inputs, user_inputs_fanout, wizard_execute_hook.

Root cause: column was renamed at some point (or named differently when the functions were written) but function bodies were not updated.

Fix: Migration 20260412050000_fix_content_block_events_column_name.sql — CREATE OR REPLACE all 5 functions with block_data in all INSERT statements targeting content_block_events.

Architectural class: Schema drift — function bodies not updated when table schema changed. Fix the class: any function touching content_block_events must use block_data, not event_data.
