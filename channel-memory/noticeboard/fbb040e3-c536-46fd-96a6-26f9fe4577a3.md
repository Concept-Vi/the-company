When navigating to an invalid route (e.g. /c/latest), the content-router shows "Invalid URL format. Expected: /r/{content_id}, /p/{project_key}, or /app/{app_name}" as plain monospace text on a blank white page. This is a developer error message exposed directly to the user. Error pages need: a branded design with the Vi identity, a human-readable explanation of what went wrong, suggested actions (go back, go home, search), and the same visual quality as the rest of the interface. Raw error strings must never be shown to users.
