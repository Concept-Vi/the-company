## 9. Things the brief doesn't mention that every mature tool eventually needed

- **Resolve / dismiss / archive a thread.** Without it the overlay fills with done items and the "needs your eyes" lane becomes noise (Docs "Resolve," GitHub "Resolve conversation"). The append-only model handles this cleanly: "resolved" is just another observation that the fold reads as closed.
- **Edit/delete a draft before submit, and edit a posted comment.** Tim already implied draft-editing; posted-comment editing is the omission. Every tool added it; the audit-trail concern is handled by keeping it an observation thread (Slack/GitHub show "edited").
- **De-dupe identical anchors.** When Tim comments twice on the same block across two sessions, surface "1 existing comment here" before he re-says it (Docs/Figma show the pin count). Cheap, high-value, and it's literally "all marks on this node" which the mark system already does.
- **A loud, never-silent failure surface.** Given the stale-anchor reality (§1) and the no-silent-failures rule: when an envelope can't re-anchor or its source moved, that must be a *visible notice + a recorded gap*, never a quietly-mis-applied edit. Hypothesis's own scar (#954: failed-to-anchor but not flagged) is the cautionary tale — the dangerous bug is not the orphan you can see, it's the mis-anchor you can't.
