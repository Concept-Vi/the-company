Deep trace found: (1) resolve_theme_inheritance merge order INVERTED — parent overwrites child instead of child overriding parent. One-line fix: swap v_resolved || v_merged_layers to v_merged_layers || v_resolved. (2) get_effective_theme overload 2 inconsistent — only fallback path uses inheritance, direct content/space themes skip it. (3) apply_theme_to_block calls get_user_theme (no inheritance) instead of get_user_theme_with_inheritance. ADDITIONALLY: compose_block returns example_value from token_vocabulary — a STATIC DICTIONARY. It never resolves actual theme colors. The theme system (887 tokens in vi_gold_global across 6 layers) is completely disconnected from rendering. Proposed fix: new resolve_component_theme(theme_id, component_type) RPC that encapsulates resolution. See: project://block-composition/rendering/theme-inheritance-deep-trace
