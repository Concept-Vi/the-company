cascade_registry has 10 entries but generate_all_from_type only executes the first 6 inline. The 4 new platform integration targets need handler functions:\n\n1. embedding_flag — when type has embedding_spec, set up embedding trigger for items of this type\n2. validation_schema — when type has data_schema, register it in component_registry for pg_jsonschema validation\n3. queue_routing — create pgmq routing rule for this type's events\n4. address_template — register ltree address template for this type's resources\n\nInfrastructure is live (pgmq queues, pg_jsonschema validate_block_content, vector columns, ltree triggers). The cascade function needs to call into them.\n\nLonger term: generate_all_from_type should READ cascade_registry and execute handler_functions dynamically rather than having all logic inline.
