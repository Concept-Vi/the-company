[paragraph] re: «So your six scopes (channel, session, project-of-the-channel, global, working-dir, "heaps") are one mechanism that already exists — it just hasn't been extended to dragnet outputs and channels yet. The ladder is built; we'd be adding "channel" and "working-dir" as new rungs and pointing dragnet cards at the right rung.»

I don’t know where you got six from, I don’t know why you are calling them mine. I don’t recognise project-of-the-channel, and the working Directory isn’t really on the same level. In my mind, a channel Belongs to a project, and if there is no specified project then that is a specific type of project, it’s an unclassified or something, like it is a global or something, that would be projects, and channels can be run without being in a project, and that is distinct by the fact that it is not one of the specific projects if that makes sense. Working directories, those are a property of projects, in the instance that those projects have content bound to working directories. The working Directory sets the relative root, I would not put that on the same page as project or channel or instance. It is related and connected of course but it’s different, a working Directory is relative to whatever is being worked on. The same Concept could be applied to a URL domain, or a database or some other addressable thing. Come to think of it I think working Directory is along one of the address axes. There are four, global project user and instance, and user doesn’t necessarily have to mean human user, like I said these are hard set names.
