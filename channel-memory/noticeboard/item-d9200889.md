  action= — channel_act only (channels.py:178). Deliberate CQRS twin of channels(op=). TENSION (naming): the read uses op= and the write uses action= for the two halves of ONE resource.
  by= — marks only (marks.py:34). The READ; the WRITE is the flat `mark` tool in server.py.
  kind= (no op) — create only (create.py:37); the enum is DERIVED at registration from Suite.create_<kind> (registry-is-truth, not a hand-listed Literal).
  verb= — session_post only (sessions.py): auto/deliver/wake/consult (a routing decision, not a CRUD op).
  binding= (BARE str) — instrument.project; discovered from registries (a Literal would go stale).

  FLAT one-verb-per-tool — server.py's ~38 module-level tools (section E).
