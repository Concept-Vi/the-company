# Breakdown · Batched submission, visibility, replies, choices

SPECIFIED: Do NOT auto-send. Compose several, submit them all with ONE action (same action whether 1 or 12). Need to SEE what's happening on screen. Need to see replies and make choices/selections if offered.

IMPLIED: A client-side DRAFT QUEUE of pending annotations held locally until a deliberate submit-all; then posted to the board/channel and routed. A live STATUS view (what's sent, what members are doing, what's come back). "Choices/selections offered" = members can pose structured questions/decisions that surface for Tim's pick (ties to the decision-surface). Implies optimistic UI + offline tolerance (phone, flaky network). Implies notification when replies arrive. Implies each draft is editable/removable before submit.

ADDITIONS: Edit/delete/reorder drafts; group or label a batch; choose target channel/members per batch or per comment; a "sent log"; undo a submission; a subtle unread-replies badge; submit from anywhere (the overlay), not only a dedicated screen.
