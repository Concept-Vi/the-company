concept_registry holds Tim's 10 canonical vocabulary definitions (Node, Slot, Socket, Skeleton, Mesh, Holon, Vi, Block, Cascade, Space) but is not surfaced by get_system_help or any MCP tool. Agents cannot discover these definitions at runtime without knowing to call query_any_table directly with table_name='concept_registry'. This means agents default to general English meanings and drift from Tim's precise definitions. The vocabulary is now stored as a project resource at project://block-composition/principles/canonical-vocabulary as a workaround, but the root gap remains: concept_registry needs an MCP-accessible discovery path, either via get_system_help (add a 'vocabulary' category) or a dedicated resolve_concept / list_concepts tool.
