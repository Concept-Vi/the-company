## 3. His MSG2 revision is the better architecture — honor it

In MSG2 Tim says he *likes the current tap-to-select*, and proposes the **long-hold be reserved for "other actions" — going more granular than the default, and opening a secondary command bar (a "right-click menu").** This is a genuinely better interaction model than his original message, and it resolves the §1 collision elegantly:

- **Normal tap = select the default/obvious block** (the paragraph, the bullet, the heading) — fast, zero-friction, the 90% case. This is *also* what dodges the native-selection war, because a tap is unambiguous and instant.
- **Long-press = "I want more control"** → arms the level-scrubber *and/or* pops the contextual command bar (multi-select, attach, change scale). The long-press becomes the *power-user escalation*, not the primary path.

This is the **iOS context-menu / `UIContextMenu` pattern** (long-press → preview + action list), which iOS users already understand from Home-screen icons, Mail, Photos. Building *on* a known platform idiom rather than inventing a parallel one is the single biggest usability win available here. **Recommendation: tap = commit default block; long-press = escalate (scrubber + command bar). Make the default-block detection good (§ the "smart" block parsing) so the tap is right most of the time and the long-press is rarely needed.**
