# Principle: the interface is not a special case

The company is already a typed, compositional system governed by established laws. The interface has been the ONE exception to those laws — a hand-coded monolith outside the architecture. That exception IS the drift. The governing principle: the interface must obey the SAME laws as the rest of the company; it is another composition in the same fabric, not a snowflake. How it joins those laws is for Tim's architecture to resolve — not prescribed here.
