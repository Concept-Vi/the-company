generate_all_from_type correctly generates MCP tool specs (create_X, list_X) but they're returned as JSON and never written to any table. All other 6 cascade targets (notice_board_types, component_registry, theme_specs, accessibility_specs, agent_decorators, capabilities) persist via INSERT. mcp_tool_registry table doesn't exist in public schema. ci_mcp_tool_registry exists in CI but has different schema. Class: cascade-target-missing-persistence — the Registration→Resolution is complete but the Connection (persisted tool definition → discoverable tool) is broken. Fix: create mcp_tool_registry table, add INSERT to generate_mcp_tools_from_type, have vi-composition-mcp read from it at startup.
