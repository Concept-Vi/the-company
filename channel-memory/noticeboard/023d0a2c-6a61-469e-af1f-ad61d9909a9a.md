Every piece of text in the interface is written for developers: "compose_block does not resolve behavioral tokens", "cascade_registry created — type cascade is now self-extending", "event_to_intent_handler references event_data but column is payload". An agnostic user cannot understand any of this. All content displayed to the user must be translated to user language: what the system did, what it means for them, what they can do about it. Technical details should be available in an expandable section for those who want it, but the primary display should communicate meaning, not implementation.
