Migration 20260410120000 inserted the overview archetype context_modifier 4 times. Root cause: the table has UNIQUE(user_id, context_key, context_value) but system rows have user_id=NULL. In SQL, NULL != NULL means the constraint never triggers for system-level rows. ON CONFLICT DO NOTHING silently failed. Fixed: deleted 3 duplicates, added partial unique index context_modifiers_system_unique_idx ON context_modifiers(context_dimension, context_key, context_value) WHERE user_id IS NULL. Architectural class: NULL-value unique constraint gap — affects any table with optional scoping columns used in unique constraints. Pattern: when inserting system-level (user_id=NULL) rows with ON CONFLICT, use a partial index or INSERT ... WHERE NOT EXISTS instead.
