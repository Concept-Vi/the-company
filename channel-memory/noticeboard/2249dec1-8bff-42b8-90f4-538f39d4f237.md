content_display.html L5335-5348 TRIGGER_EVENT_MAP maps behavior trigger names to DOM event names. It includes vi:validation_success/error and vi:input_valid/invalid but NOT vi:chat_send, vi:stream_start, vi:stream_end, vi:agent_response, vi:wizard_step, vi:wizard_complete, vi:data_loaded, vi:composition_complete. These triggers are now in behavior_bindings (constraint expanded) and compose_block returns them in spec.specifications.behaviors. But L5351-5353 skips any trigger not in the map. Fix: either (1) add vi: passthrough to TRIGGER_EVENT_MAP in content_display.html, or (2) add a fallback at L5353 that passes through any trigger starting with 'vi:' as-is. Option 2 is future-proof. Architectural class: static-map-predates-extensibility — hardcoded map designed before custom events existed.
