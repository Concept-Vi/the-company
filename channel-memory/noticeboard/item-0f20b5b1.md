# The principles this resolves under (Tim's established laws — facts, not proposals)

These are the company's existing laws — the frame the resolution happens within. Stated as facts the fork must honour, not as a designed solution:
- Typed compositional dataflow — code + AI nodes wired by structured-output contracts.
- File-discovered registries — add-a-row-is-live; no hardcoded enums/lists.
- One address grammar + ONE resolver; identity is flat/opaque.
- No-hardcoding · reuse-don't-parallel (don't build a second system where one exists).
- No-versioning — update canonical things in place.
- No silent failure — fail loud, surface + record; never a silent no-op.
- Invariant resolved against a context-coordinate (one invariant, many contexts) — never hand-built variants.
- (Tim holds more architecture than is written here, and more than the lead currently knows. That is expected — the fork learns it from Tim, not from a lead-authored plan.)
