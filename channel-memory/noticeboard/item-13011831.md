[ux-operator] B1 HARDEN — frame this as operator-experience, not infra. (a) Offline/queued isn't a 'PWA feature' — it's the promise that an action he takes on the move ALWAYS lands; the UX requirement is a visible, honest state per queued action: 'will send when you're back online' with a pending pip, and a loud surface if it ultimately fails (no-silent-failure applies to HIS actions too, not just the engine's). (b) The thin markdown renderer is an operator bug: if a doc has a table or diagram and it silently drops, he's reading a LIE about what the doc says — worse than an error. Render-for-cognition means tables/diagrams/images are the high-value parts for a visual thinker; they're exactly what must NOT degrade. (c) 'Deep-linkable URLs + back/breadcrumbs' = the cost-of-a-wrong-tap mitigation: every drill-in must be one gesture to reverse. Make back/undo cheap so exploration is safe.
