# The work-movers — a layered stack, all threading via run:// addresses

- **PRIMITIVES** (`cognition.py`): run_role (1→1) · run_items (1 role × N units, fan) · run_reduce (N→1 join; modes role/rule/cluster) · run_jury (N varied draws→deterministic verdict) · run_panel (1×N lenses→quorum) · run_swarm (N roles concurrent). = map / fan / join / compose.
- **CASCADES** (`coherence_actions.py` validate, `cognition.py:run_cascade`): declarative SAVED pipelines (`cascades.json`); steps thread output→input by step-index; 7 step-kinds (role/items/reduce/retrieve/check/jury/panel); the runner owns persist+index. 8 live (verify-jury, option-panel, spec-compiler, ask-the-codebase…).
- **FLOWS** (`flows.py` + `flows/*.py`): committed **PROPOSE-ONLY** production lines (file-discovered, `proposes_only=True` enforced at discovery); stage data + grounding, invoke cascades/primitives, resume-safe, bounded. 8 live (registry_generation, repo_ingest, drift_radar, floor_walk…).
- **COMPOSITIONS / MINDS** (`minds.py` + `minds/*.py`): ordered role-comps wired by typed feeds-edges; recompose = edit a row, zero code.
- **SCHEDULER** (`scheduler.py`): reactive canvas-DAG executor (fires a node when its input addresses resolve; serial BY DESIGN).

Hierarchy: **FLOW (stages) → CASCADE (declares) → PRIMITIVES (execute).** LAW: *"the proven chain is the easiest path"* — flows exist so agents reuse grounded pipelines instead of rebuilding ungrounded ones.
