mode_registry has 10 modes each with display_name, description, and prompt_fragment: companion (general assistance), creative (generation/ideation), diagnostic (troubleshooting), discovery (pattern detection), emergency (crisis response), execution (task completion), facilitation (coordination), onboarding (user guidance), research (deep investigation), teaching (education). The mode_selector component is registered and the Settings view shows radio buttons for modes, but with no visual design. Each mode should have an icon, a color accent, and a description of what it means for the user's experience — not developer labels. The active mode should visually influence the interface (context modifier system supports this).
