Between prototype and daily-reliable:
- **Durability** — it's an ephemeral background process that dies on reboot → a `company-surface` systemd service, auto-restart, the deliberate 8790 flip.
- **Offline + queued actions** — a real PWA service worker: cached docs/artefacts readable with no signal; comments written offline queue and send on reconnect (Tim is mobile, on the move).
- **Performance** — replace the full-board re-scan per request with an in-memory index + incremental update on write.
- **Real content rendering** — the markdown renderer is a thin subset: NO tables, NO code blocks, NO inline images, NO diagrams. Documents are the weak side (artefacts render fully via iframe).
- **The unglamorous floor** — loading/empty/error states, deep-linkable URLs, back/breadcrumbs, font-scaling, focus states.
