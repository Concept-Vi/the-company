### Option D — Vault root = `channel-memory/noticeboard/` (max feature, max risk)
Point Obsidian directly at the canonical dir. Zero projection plumbing — the files Obsidian sees *are* the
truth. Maximum feature payoff with the least code.

- **This option is CONDITIONAL and currently UNVERIFIED.** It is only viable if Obsidian's editor/Sync
  **preserves the nested `links: [{kind,target}]` and `history:` arrays byte-faithfully on save.** It is a
  known, documented weakness that Obsidian's Properties UI renders arrays-of-objects as "unsupported" and
  its YAML serializer reorders keys and can drop/flatten complex nested frontmatter. Our real files
  (verified §0) carry exactly those nested arrays. **If Obsidian mangles them, Option D destroys the edge
  set and the lifecycle history — it is out.** Do not present D as equal to A/B/C until this is tested on
  a throwaway copy of a real `<id>.md`.
- Additional bites even if YAML survived: `.obsidian/` config litter inside the canonical git-tracked
  dir; Obsidian's serializer churning `sort_keys` order against our `yaml.dump(sort_keys=False)` →
  noisy git diffs and round-trip instability; and Obsidian Sync acting as a *second* sync system with its
  own schema-blind conflict resolution layered over git (see §6).
