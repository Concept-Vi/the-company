## The single capability that makes the rest optional

If I could ship exactly ONE improvement, it is this:

> **Batched submit + a visible reply inbox on the phone.** Compose several comments, hit submit-all once, and see replies come back without leaving the page.

Why this one and nothing else: it closes the *loop into a cycle*. Right now (single comment, reply in place) Tim is doing one transaction at a time and has to go hunting for replies. The batched draft-queue + a reply badge turns the artefact from a comment box into a **working session**: read the doc, drop ten notes, fire them, watch answers land. That is the actual behaviour Tim described wanting ("I might make a dozen... submit them all... see what's happening... see replies"). Everything else in the brief is in service of this and most of it isn't needed to get it.

Nail this, and the gesture system, the UI mode, the document tree, the radial menu, the attachments — all become *nice-to-have polish on a system that already cycles.*
