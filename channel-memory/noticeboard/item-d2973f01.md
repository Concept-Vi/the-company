# Convergent improvements (multiple lenses agreed)

1. **Bind to existing machinery, don't build a fleet** — run the cheap no-judgment tier on the resident engine; author the whole process as a registered, resume-safe *flow* (like the proven registry-filling one). [strongest — 3+ lenses]
2. **Drop "agents discuss as channel members on a channel"** — there is no automatic path for that (the safety system blocks an agent from doing it). Use the **file/board hand-off** the company already runs on.
3. **Freeze the comment set first** — snapshot a numbered, hashed, timestamped list of all your comments; make "every one accounted for" a **blocking gate**, not a late check.
4. **Audit the map before it shapes the run** — the cluster-map gets its own check (and can send work back), so a wrong map can't mis-configure everything downstream.
5. **Discover the critic lenses (with a floor), seeded from the 7 perspectives you already have** — resolves a contradiction in the plan about who picks the lenses.
6. **Capture the IMPLIED, not just the facts** — the extractor must capture facts + implied + additions (labeled); your brief weights implied > literal, and "facts only" would throw away your highest-value layer.
7. **Write-discipline** — many agents write at once and the board has no locking, so workers only file NEW items (no collisions).
8. **Use the real change-wire's real name** — the "pipeline" the plan names has no code on main; point at the one that exists.
9. **Tier the context by role** — extractors get a minimal anchor + their one comment-with-content (not the firehose); the judges get the full frame.
10. **Split the forks into two lanes + your checkpoint** — reversible engine/infra decisions proceed on a default; taste/direction forks wait for you.
11. **Make the run a gated loop, not a one-way conveyor** — audit findings can send work back to re-cluster or reopen a room.
12. **Map room dependencies as a graph** — sequence dependent rooms; parallelise only the independent ones.
