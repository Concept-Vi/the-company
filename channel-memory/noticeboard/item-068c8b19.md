# The dragnet as graph + the "one wire"

The dragnet emits three graph-shaped things:
- (a) **field-index** = a real typed triple-store `{target: code://…, field, value}` (11.5k rows, `ops/code_archaeology.py`) — deterministic, the MOST precise output, but **UNTRAVERSED** (flat CLI lookup only).
- (b) **extractions** = embedded + addressed (`extraction://`) + k-NN traversable (`corpus` op=neighbours/determine).
- (c) **board traversal** exists (`cc_board`) but only scans board items.

**THE ISOMORPHISM (the crux):** `query_field_index(field, value)` IS structurally `reverse_traverse(subject=?, predicate=field, object=value)`. The field-index is a triple-store the traversal engine can read the moment it's wired — not a missing abstraction, a missing connection.

**THE ONE WIRE = 3 pieces, smallest-first:**
- **A (keystone, ~20 lines):** add `code://` resolution to `resolve_address` (mirror the `board://` branch) → code nodes become first-class.
- **B (3 tiny files):** register `declares` / `imports` / `symbol` as `relation_types/`.
- **C (~40 lines):** wire `reverse_traverse` to read the field-index for those kinds.
Wire A alone is testable & unblocks board-items linking to code.
