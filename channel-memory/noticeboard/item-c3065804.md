# Background: what exists today (scaffolding)

So the fork has the real picture (not to copy, to understand): the current interface (served at the tailnet, on Tim's phone) contains — a document/comment review surface (board documents rendered as addressable, commentable blocks; comments carry a typed envelope: target address, scale, the tapped text, author); a realtime two-way CHAT (operator ⇄ lead) over the channel fabric (the operator app registered as a pushable channel member; SSE pushes to the browser; a responding indicator; image attach); a hamburger document switcher; live-animated auto-reload on deploy. It is hard-wired to ONE channel. All of it is a fast-built monolith. It WORKS — it is the proof, and the material to be re-homed properly — but it is not built under the architecture.
