Frontend queries ci_lens_registry: GET /rest/v1/ci_lens_registry?select=lens_id,lens_name,primary_index,enabled → 403

OBSERVED: ci_lens_registry has RLS policy: service_role only (ALL commands). Authenticated user token cannot read this table.

Fix: Either (a) add a SELECT policy allowing authenticated users to read ci_lens_registry, or (b) remove the frontend query — CI lens data should not be loaded by the project display renderer.

The ci_lens_registry is part of the CI (Conversation Intelligence) system, not the composition pipeline. The frontend should not query it directly.

Recommended fix: Remove the ci_lens_registry query from project_display.html. If lens data is needed for the graph view, expose it through a composed block or a dedicated MCP tool that runs with elevated permissions.
