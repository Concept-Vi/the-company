The Settings view shows native browser radio buttons with plain text labels (Detailed, Brief, Balanced, Creative, Investigative, etc.). No visual framing, no card containers, no section headers with proper typography, no description of what each setting does in user-appropriate language. The Vi Mode section shows mode names but doesn't explain what each mode means for the user's experience. Settings should be presented as designed cards with icons, clear descriptions, visual selection states, and grouped into meaningful sections with proper hierarchy.
