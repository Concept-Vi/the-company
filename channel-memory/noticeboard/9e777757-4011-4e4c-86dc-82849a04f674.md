When compose_block fails to resolve a component, or a token is missing from the cascade, or a template has unresolved variables, the user sees nothing. The composition pipeline is supposed to surface diagnostics that say WHAT is missing and WHERE to fix it. These diagnostics exist in the pipeline output but are not rendered. When something fails to compose, the user should see a diagnostic card with: what was requested, what failed, what is missing, and a suggested action. Silent failures hide the system's gaps.
