project_display.html uses hardcoded @media (max-width: 768px) for layout changes. But spatial context_modifiers already have token_overrides for device/surface/accessibility: sys-layout-max-width (1280px desktop, 1024px tablet), sys-spacing-container (24px/20px), sys-mobile-min-touch-target (56px), sys-mobile-safe-padding-top/bottom, ref-font-size-base scaling. Block-level rendering reads these through compose_block → ThemeEngine. Page-level layout should too. Class: page-layout-bypasses-block-cascade. Fix: boot() should resolve spatial context (device detection already in CURRENT_CONTEXT), get token_overrides from context_modifiers, inject as CSS variables on body, and layout CSS should reference --sys-layout-max-width etc instead of hardcoded breakpoints.
