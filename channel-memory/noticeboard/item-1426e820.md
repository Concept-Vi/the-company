## 7. Conventions Tim already uses (don't reinvent these)

- **Note naming:** `(Type) Title.md` — e.g. `(Term) Seed Kernel.md`, `(Code) ... .md`.
- **Frontmatter schema:** `id` (YYYYMMDDHHMMSS), `type`, `created_date`, `updated_date`, `processing_status` (inbox|skeleton|processing|processed|evergreen|archived). The company-repo variant uses `type/module/aliases/tags/governs/relates-to/status`.
- **Aliases-not-filenames for links** (because every constitution is `AGENTS.md`).
- **Em-dash ` — `** as the aesthetic separator in note titles/aliases.
- **`.base` views grouped by a status field, with formula columns** — the established "board" rendering idiom (Dashboard.base, Status Board.base).
- **Spaces/** as a top-level grouping (Obsidian Builder has `Spaces/Mobile Parity/...`).
- **Canvas for architecture maps** kept current by agents.
