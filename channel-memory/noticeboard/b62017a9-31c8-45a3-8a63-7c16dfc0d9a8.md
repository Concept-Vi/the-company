OBSERVED in renderSettings (session 15):

modeBlock.content.modes array uses field: mode_name
mode_selector template uses: {{mode_key}} in data-mode-name attribute and action_payload.token_value

Effect: {{mode_key}} evaluates to empty string → data-mode-name is empty → dispatchAction guards against empty token_value and exits early → mode selection falls through to post-render wiring which uses mode.mode_name directly (bypassing pipeline).

FIX: In renderSettings, rename mode_name → mode_key when building modeBlock.content.modes:
  mode_key: m.mode_name,  (or rename in DB: mode_registry.mode_name → mode_registry.mode_key)

One-line fix in project_display.html renderSettings around line 5406. Does not require migration if keeping mode_registry.mode_name column — just remap at the block-building step.
