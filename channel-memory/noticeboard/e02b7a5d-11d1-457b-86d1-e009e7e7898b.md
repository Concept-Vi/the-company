When views are loading, there is no loading indicator or skeleton screen. When a view has no items (like ACTIVE column with 0 items), it just shows "No items" as plain text. When something fails to load (like the graph data), there is no error state. Every state of every component needs a designed treatment: loading (skeleton shimmer or spinner), empty (illustration + guidance text explaining what will appear and how to create content), error (clear message + retry action). These states are part of the experience quality.
