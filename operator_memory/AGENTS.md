# operator_memory/ — module constitution

**Is:** the **file-discovered OPERATOR-MEMORY registry** (GC15) — the system's growing, defined memory
of working with Tim. One row = one known thing (a rule · preference · standing approval · articulated
principle) with his VERBATIM EVIDENCE attached. This is the memory ALL agents get — the RHM, crew,
and the coldest external agent through the MCP — not any harness's private notes. Born from Tim's
call (2026-06-10): "having growing memory of me in the system is a major feature value that currently
doesn't have any real definition."

**The lifecycle:** the transcript mining PROPOSES rows when patterns recur (status='proposed');
TIM'S CONFIRMATION makes them standing (status='confirmed' + the `confirmed` record). The same
propose→confirm circuit as everything else. Evidence is REQUIRED — a memory of Tim without his words
is fabrication (fail-loud at discovery).

**Scoping:** rows may declare `scope.when` (the activity it matters most during — the GC14
condition-clause shape) and `scope.where` (a ui:// prefix). Consumers filter by it today; the GC14
self-injection mechanism is the designed next layer.

**Loader:** `runtime/operator_memory.py`. **Face:** `mcp_face/tools/operator.py` (op=rules|describe|proposed).
**Floor:** pure data; nothing executes. **Never:** a row without evidence · auto-confirm a proposed
row · fork a second Tim-knowledge store (this registry IS the definition).
