# checks/ — module constitution

**Is:** the **file-discovered CHECK registry** (G3·S3a) — deterministic gates as declared data. One
row = one pure verdict function ({passed, reasons}; same value, same verdict — no model, no time).
The generalization of the recurring arc: refcheck → prose_check → the radar's relation filter — when
a model judgment misfires on a rule-shaped constraint, the rule becomes a registry row, referenceable
by NAME from cascade decls ({op:'check', check:'<id>'}). Loader: `runtime/checks.py`.

**The law:** a check JUDGES a value — never resolves/dispatches; its verdict ROUTES chain units
(annotate by default; declared `on_fail:'drop'` records every dropped unit — never silent).
Add a check = add a FILE + reflect it here.
