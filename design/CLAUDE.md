## What you can do here (the operations)
- **Change the look from one place** ‚Äî edit `tokens.json` ‚Üí `python3 _system/emit.py` ‚Üí every mockup reflows. (Shared values are a `primitive` once, `ref`'d everywhere.)
- **Address + map** ‚Äî carry `data-ui-ref="ui://‚Ä¶"` on meaningful elements, register them in `addresses.json`, `python3 _system/parse.py` ‚Üí the element ‚áÑ feature ‚áÑ code map + orphans (both ways).
- **See what's missing** ‚Äî `coverage` (features with no view) and the parser's *registered-but-unused* list (addresses not yet placed) ARE the to-do. `python3 _system/check.py` rolls these up + finds hardcoded literals that should be tokens (matches an existing token ‚Üí use `var`; recurs with no token ‚Üí candidate new token), coverage gaps, orphans, and hygiene ‚Äî the free deterministic floor, before any model runs.
- **Refresh the gallery** ‚Äî after adding/addressing a mockup or editing `register.json`, `python3 _system/gallery.py` regenerates `index.html` from the register + the real files.
- **Generate grounded mockups** ‚Äî register ‚Üí a writer grounded in the inventory + design-system ‚Üí render ‚Üí quality-gate (grounded? on-system? addressed? no fiction?) ‚Üí gallery.
- **Catch traceability drift** ‚Äî `python3 _system/refcheck.py` resolves every `code:` ref in `register.json` (`features[].code`) + `addresses.json` (`addresses.*.code`) against the real source under `~/company` (**READ-ONLY**) and reports where a ref no longer lands on the symbol it claims (with a repair target where it can find one) ‚Üí `refcheck-report.json`. The refs drift as the app grows; this is the free deterministic floor that catches it. The keeper/lead repairs the registries **centrally** (never edit them mid-analysis ‚Äî races).
- **Index code symbols (the `code://` branch)** ‚Äî `python3 _system/symbols.py` builds the **reverse** index of the same refs: every code symbol the corpus references becomes an addressable entity keyed by `code://<file-stem>/<symbol>` (e.g. `code://suite/resolve_surfaced`) ‚Üí `code-symbols.json` = `{ file ¬∑ symbol ¬∑ kind(def|class|route|const|file-only) ¬∑ resolves ¬∑ referenced_by[] }`. Where `refcheck` is forward (ref ‚Üí does it resolve), this is symbol ‚Üí **who references it**; `resolves:false` is a drift signal, `referenced_by` with 2+ owners is a **shared symbol** (a change there ripples to several features/addresses). **X11 ¬∑ the SEMANTIC edge:** each entry ALSO gains `semantically_nearest[]` (`{id, score}`, top-`SEMANTIC_K` by cosine) ‚Äî symbols **conceptually related with no code link**, the SIBLING of the structural `referenced_by[]` (the complement of `codeedges.py`'s call-graph). Built by **reusing** `nodes/embed` (‚Üí BGE-M3 @ `:8001`) + `nodes/retrieve` (cosine ranking ‚Äî not reimplemented): each entry's representative text (symbol + kind + file + its owners' `represents`) is embedded ‚Üí the rest ranked ‚Üí top-`SEMANTIC_K` kept. **Degrades-with-warning when `:8001` is down** (the field is skipped for all entries with a loud warning ‚Äî never a fabricated nearest; the structural index still builds fully). Realises the seam this script's docstring reserved. Regenerate after any `code:` ref change. Gives the future local-AI layer one canonical symbol list to do semantic checks against.
- **Map code dependencies (the symbol‚Üísymbol graph)** ‚Äî `python3 _system/codeedges.py` builds the **structural code-dependency graph** (Convergence X10): which symbol *calls/imports* which. The reverse-index (`symbols.py`) gives corpus-artifact‚Üísymbol; this gives **symbol‚Üísymbol** ‚Äî the relational layer that existed nowhere. It parses the real Python source under `~/company` (**READ-ONLY**, stdlib `ast`) ‚Üí `code-edges.json` = `{ code://id: { depends_on[] (dependencies-to-RESPECT) ¬∑ depended_by[] (dependents-to-VERIFY) ¬∑ resolves } }`, keyed by the **same** `code://` ids `symbols.py` uses (a **sibling**, never a parallel). Only **direct** edges are stored; the transitive reach is a **bounded** query (`codeedges.reach` / `reach_report`) capped at the named `DEPTH` (2‚Äì3 hops per Tim, X17-configurable via `CODEEDGES_DEPTH`, default 2) ‚Äî the explosion guard, with a capped reach reported (no silent truncation). An unparseable source file ‚Üí a `stale[]` entry (`resolves:false`, **fail-loud**), never dropped. This is what the blast-radius (X14) reads for the structural dependents/dependencies of a pointed self-build. Regenerate after the `~/company` source changes.
- **Mechanisms (the analysis registry)** ‚Äî corpus-analysis checks are registered in `_system/mechanisms.json` (keyed by `id`): each reads part of the corpus, emits findings, routes them (repair | surface). This is **Layer-0** (exact, free, no models); a local-model layer augments each later (semantic judgement on top of structure ‚Äî e.g. *does the symbol actually implement what the corpus claims*). Add a check = register a mechanism + drop its `_system/<x>.py` + `test_<x>.py` (same shape as `check.py`/`refcheck.py`). The mechanism-registry generalises the `check.py` floor into an extend-by-registration set of checks.
- **Local-swarm moves** (cheap concurrent local models) ‚Äî recognise-and-propose (find a recurring literal ‚Üí propose a token), hardcoded‚Üítokenised passes, design-lint, en-masse analysis. Read-against-a-reference, at scale. *(The model layer that augments the mechanisms above.)*
- **Extend-by-registration** ‚Äî add a token / address / invariant / view / mechanism by *registering an entry* and running the script. Never surgery.
- **The horizon** (design-reasoning, see the substrate) ‚Äî the corpus-as-graph you can query; the convergence to real components.
