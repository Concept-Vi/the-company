The page marker — then & now

Now — generated glyphic
Hexagon + palette icon → dashed gold arrow → square + browser icon. Typed + iconned + dimensional — but the pill, the “solid” feel, and the label are gone.

Your idea — the node’s FILL carries the meaning

Same box, three fill-states, three meanings. (“present / type / potential”.)

Solid fill, no icon — “this thing, here, present.” Pins to anything; the generic marker you liked.
Outline only — “any of this kind.” A type / class, not a specific instance.
Dashed outline — “a thing of this kind could be added here.” A potential / open slot.

The edge carries meaning too

The line-style and the arrow direction are facets, same as fill.

Dashed → — a potential path: “can go to / can become.”
Solid → — an actual, asserted relationship that holds right now.

Same three things — the value changes the meaning

Two nodes + one edge. The page relation, read three ways.

this thing here can link to / become a page
this thing IS the face of a page (asserted, live)
a page could be added here (both ends potential)
To make this real, Glyphics needs parts it doesn’t have yet: a solid-colour fill + an icon-optional node (the “this thing” box) · a ring stroke-style facet (solid vs dashed = present vs potential) · and the edge joining the meanings registry so line-style + direction mean by profile, like fill already does. Then the engine generates every reading above from facet values — meaning = f(source · edge · target).
