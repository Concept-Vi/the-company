The Type System

Everything on an interface is a universal component — a typed thing that declares its parts, its value-slots (parameters that take an axis value), and its sockets (typed attachment points where other components, or events, plug in). This page is a live projection of window.CV_REGISTRY: the Glyphic, all 11 UI components, the panel / graph / slide kinds, and the block/surface/doc library — one grammar, total coverage. Acceptance is declarative, so candidatesForSocket() shows what fits a socket with no bespoke code. Model: system/glyphic-system.html.

Single-inheritance lineage — a Type extends a parent and inherits its parts, slots and sockets (flattened by CV_REGISTRY.resolve). These are the trees where that relationship is live. This is the home for type inheritance; the file map shows file resolution, not type lineage.
