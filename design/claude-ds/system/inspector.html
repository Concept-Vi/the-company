ConceptV · Living Projection

The One Inspector

The spec asks for it repeatedly: build the socket inspector once over CV_NODE.candidates, so the same panel edits a token, a glyphic, a panel, or a slide. This is that one module — window.CV_INSPECTOR. Each column below is the identical CV_INSPECTOR.mount() call on a different kind; it reads CV_PROJECT.toInspector(node) and renders the whole projection — value-slots, the sockets half (previously invisible everywhere in the UI), decorators and actions — with every option previewed by rendering the node through CV_NODE.render. No per-kind code. Edits go through the one action layer (CV_ACTIONS.invoke) — the same verbs the AI calls.
