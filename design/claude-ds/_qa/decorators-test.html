CV_DECORATORS — the @decorator vocabulary, proven

One catalogue of every cross-cutting annotation. It is retrospective: each decorator derive(node) reads the value from the node's existing field — nothing is copied to a second home. Decorators can find (search by annotation), gate (a condition), generate (route to a CV_AI capability), edit (name an inspector's fields) and accept (a socket can require/forbid them).
