# DEEP COMPARATIVE PASS — what the two cuts actually *do* differently
> The diff between the cuts is itself data. Reading it reveals principles neither deck shows alone.

## A · Space ↔ time is a complexity trade-off (the biggest one)
The clearest tell: pitch-deck shows "user-relevant layers" as a **static stack of overlapping panels** (complexity rendered in *space*); deck1-2026 replaces it with **one clean image + a GIF** (the same complexity rendered in *time*). → **A slide has a fixed complexity budget that can be spent in space (static, simultaneous) OR time (animated, sequential).** Density and motion are not separate features — they're two ways to pay the same bill. Design implication: a "dense static diagram" and a "simple frame + animation" are *register variants of one component*, swappable.

## B · Who controls the pace determines the register
- **Leave-behind (pitch-deck):** the *reader* controls the timeline — can re-read, pause — so prose is dense and **self-sufficient** (works with no human present).
- **Presented (deck1-2026):** the *presenter* controls the timeline and speaks the detail, so the slide carries only **anchors** (headline + number + visual) and the GIF animates while they talk.
→ Register isn't a style preference; it's set by **who paces the artifact**. Predictive rule: a self-serve/mobile surface needs the *verbose* register (no presenter to fill gaps); a live pitch needs *terse*. The register switch should be driven by "who's pacing it", not picked arbitrarily.

## C · Level-of-Detail is an axis orthogonal to surface size
The two decks are the same message at two **zoom levels** — like map LOD. A one-pager would be a third, tighter zoom. → The system isn't only **multi-surface** (screen sizes); it's **multi-LOD** (detail levels), and the two axes are *independent* (a big screen can show low detail; a phone can show high detail via scroll). Templates should expose **content LOD** as its own control alongside surface. This is what lets one source generate a 17-page read, a 16-page pitch, and a 1-page summary.

## D · The invariant skeleton vs the flexible muscle
What **never changes** between cuts: the diagrams, the **numbers** (`$430B`, `70+`, `41,500+`), the archetype, the frame signature, the colour/zoning. What **flexes**: prose length, title wording, slide count. → The deck's *skeleton* = **quantitative evidence + visual structure** (durable, never cut); its *muscle* = **prose** (register-variable). Design implication: templates should treat numbers/diagrams as **fixed payload slots** and prose as **LOD-variable text** — never the reverse. Numbers are sacred; sentences are negotiable.

## E · Every bullet decomposes to {claim, support}
The verbose→terse diff is a **compression map**: "The design, construction and sale of new property takes numerous people…" → "**Numerous stakeholders**". The terse cut keeps the *claim* and drops the *justification*. → Model each bullet as **{claim, support}**; render claim-only (terse) or claim+support (verbose) by LOD. This is a far more precise content-model than "fewer words".

## F · Motion has placement rules (it's not sprinkled)
The GIF placeholders sit only on **(a) the hero concept diagram** and **(b) immersive product/spatial views** (virtual tour, room walkthroughs) — never on analytical/stat slides. → Motion vocabulary placement rule: **animate the concept and the product-experience; keep analytical slides still.** A guardrail for the motion language so it stays meaningful.

## G · Slides carry a narrative-priority rank (enables auto-shortening)
17→16: a slide was trimmed (the SaaS/E-commerce triptych) while problem/solution/ask survived. → Slides have an implicit **essential vs optional** rank; the presented cut drops optional ones for time. Tagging archetype instances with narrative-priority would let one deck **auto-shorten** to a time budget.

## H · "Ink budget" is conserved; register redistributes it
Terse cut = fewer bullets, **larger** type; verbose = more text, smaller. The text *area* stays ~constant — register reallocates it between "more words, smaller" and "fewer words, bigger". → Reinforces A: a conserved budget, spent differently. Useful sizing rule for the register switch (don't just delete words — resize the survivors).

## I · Which file is canonical
deck1-2026 is **2× resolution, named "2026", motion-ready** ⇒ the **living master**; pitch-deck is an older/derived export. → When values conflict, **sample from deck1-2026**; treat it as source-of-truth for production.
